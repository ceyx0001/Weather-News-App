\doxysubsection{Detailed Description}
Exception thrown when Wt\+::Dbo detects a concurrent modification 

Wt\+::Dbo uses optimistic locking for detecting and preventing concurrent modification of database objects. When trying to save an object that has been modified concurrently by another session, since it was read from the database, this exception is thrown.

This exception is thrown during flushing from \mbox{\hyperlink{classWt_1_1Dbo_1_1Session_ab896d119f1e0fb79adffe2282b92ef17}{Session\+::flush()}} or \mbox{\hyperlink{classWt_1_1Dbo_1_1ptr_a3e4e12079f2e3595bc201207217b6a4f}{ptr\+::flush()}}. Since flushing will also be done automatically when needed (e.\+g. before running a query or before committing a transaction), you should be prepared to catch this exception from most library A\+PI calls.
