\doxysubsection{Detailed Description}
A rich-\/text X\+H\+T\+ML editor. 

The editor provides interactive editing of X\+H\+T\+ML text. By default it provides basic mark-\/up (font, formatting, color, links, and lists), but additional buttons may be added to the tool bars that add additional formatting options.

The implementation is based on \href{http://tinymce.moxiecode.com/}{\texttt{ Tiny\+M\+CE}}. The widget may be configured and tailored using the \mbox{\hyperlink{classWt_1_1WTextEdit_a6abdda3e80332ee95223ab0c364d72ee}{set\+Configuration\+Setting()}} and related methods that provide direct access to the underlying Tiny\+M\+CE component.

\doxyparagraph*{C\+SS}

Styling through C\+SS is not applicable.

This is used by the layout managers to correct for a built-\/in border which interferes with setting a widget\textquotesingle{}s width (or height) to 100\%.

For form widgets, the border width depends on the specific browser/platform combination, unless an explicit border is set for the widget.

When setting an explicit border for the widget using a style class, you will want to reimplement this method to return this border width, in case you want to set the widget inside a layout manager.

{\footnotesize\ttfamily virtual void Wt\+::\+W\+Text\+Edit\+::resize (\begin{DoxyParamCaption}\item[{const \mbox{\hyperlink{classWt_1_1WLength}{W\+Length}} \&}]{width,  }\item[{const \mbox{\hyperlink{classWt_1_1WLength}{W\+Length}} \&}]{height }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [override]}, {\ttfamily [virtual]}}



Resizes the widget. 

Specifies a fixed size for this widget, setting C\+SS {\ttfamily width} and {\ttfamily height} properties. By default a widget has automatic width and height, which sets a size for the widget following C\+SS rules.

When the widget is not managed by a layout manager, the automatic (natural) size of a widget depends on whether they widget is a {\itshape block} or {\itshape inline} widget\+:
\begin{DoxyItemize}
\item a {\itshape block} widget takes by default the width of the parent, and the height that it needs based on its contents
\item an {\itshape inline} widget takes the width and height that it needs based on its contents (possibly wrapping over multiple lines). The width and height of an inline widget cannot be changed (by the letter of C\+SS, although most browsers will react to it in varying ways).
\end{DoxyItemize}

When inserted in a layout manager, the size set will be used as a widget\textquotesingle{}s preferred size, but the widget may be given a different size by the layout manager based on available space and stretch factors. The actual size given by a layout manager may be retrieved by making the widget \char`\"{}layout size aware\char`\"{}, using \mbox{\hyperlink{classWt_1_1WWidget_a4f83592912a7f8fa4fd35dadde78ee74}{set\+Layout\+Size\+Aware()}}. If you have defined a {\ttfamily \char`\"{}wt\+Resize()\char`\"{}} Java\+Script method for the widget, then this method will also be called.

The default width and height of a widget is \mbox{\hyperlink{classWt_1_1WLength_a0cf39ca4225776879d56ade60320c31a}{W\+Length\+::\+Auto}}.

The syntax and available buttons is documented in the \href{http://www.tinymce.com/wiki.php/Configuration:toolbar\%3CN\%3E}{\texttt{ Tiny\+M\+C\+Edocumentation}}.

The default {\itshape config} for the first tool bar ({\ttfamily i} = 0) is\+: \char`\"{}undo redo $\vert$ styleselect $\vert$ bold italic $\vert$ link\char`\"{}.
