\doxysubsection{Detailed Description}
A paint device for rendering to a P\+DF. 

A W\+Pdf\+Image paint device should be used in conjunction with a \mbox{\hyperlink{classWt_1_1WPainter}{W\+Painter}}, and can be used to make a P\+DF version of a \mbox{\hyperlink{classWt_1_1WPaintedWidget}{W\+Painted\+Widget}}\textquotesingle{}s contents.

The P\+DF is generated using \href{http://libharu.org/}{\texttt{ The Haru Free P\+DF Library}}, and this class is included in the library only if {\ttfamily libharu} was found during the build of the library.

You can use the image as a resource and specialize \mbox{\hyperlink{classWt_1_1WPdfImage_a45ef6cac7be6a5ecfa0d060db0d9ad5a}{handle\+Request()}} to paint the contents on the fly. Alternatively can also use \mbox{\hyperlink{classWt_1_1WResource_aaf6d964de1056c7c9404f88fa5e5a0de}{write()}} to serialize to a P\+DF file (std\+::ostream). The latter usage is illustrated by the code below\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{ \mbox{\hyperlink{classWt_1_1Chart_1_1WCartesianChart}{Wt::Chart::WCartesianChart}}}
\DoxyCodeLine{*chart = ...}
\DoxyCodeLine{}

\DoxyCodeLine{\{}

\DoxyCodeLine{  chart-\/>paint(p);}
\DoxyCodeLine{\}}
\DoxyCodeLine{std::ofstream f(\textcolor{stringliteral}{"chart.pdf"}, std::ios::out | std::ios::binary);}
\DoxyCodeLine{pdfImage.write(f);}
\end{DoxyCode}


A constructor is provided which allows the generated P\+DF image to be embedded directly into a page of a larger {\ttfamily libharu} document, and this approach is used for example by the W\+Pdf\+Renderer to render X\+H\+T\+ML to multi-\/page P\+DF files.

Font information is embedded in the P\+DF. Fonts supported are native Post\+Script fonts (Base-\/14) (only A\+S\+C\+I\+I-\/7), or true type fonts (Unicode). See \mbox{\hyperlink{classWt_1_1WPdfImage_a479b8988ba6d379088ef097b894cbc22}{add\+Font\+Collection()}} for more information on how fonts are located and matched to \mbox{\hyperlink{classWt_1_1WFont}{W\+Font}} descriptions.

This paint device has the following limitations\+:
\begin{DoxyItemize}
\item images (\mbox{\hyperlink{classWt_1_1WPainter_ae36e5ce0c6a324ec83ebbc397a513202}{W\+Painter\+::draw\+Image()}}) can only be included from local files, and only J\+PG and P\+NG images are supported.
\item drop shadows are not supported. 
\end{DoxyItemize}

The single page will have a size {\ttfamily width} x {\ttfamily height}. The P\+DF will be using the same D\+PI (72dpi) as is conventionally used for the desktop.

The passed width and height (such as 4 cm by 3 cm) can be specified in physical units (e.\+g. 4cm x 3cm), but this will be converted to pixels using the default D\+PI used in C\+SS (96dpi) !

You may need to add more glob patterns to exclude other fonts than True\+Type, and also to exclude True\+Type fonts which do not work properly with libharu.

If Wt has not been configured to use {\ttfamily libpango}, then this method may be used to indicate the location of True\+Type fonts. The main drawback compared to libpango is that font selection is not steered by the need for particular characters, i.\+e. font selection is independent from the text\textquotesingle{}s need for specific characters. Most True\+Type fonts provide only partial unicode support. The provided {\ttfamily directory} will be searched for fonts (currently only True\+Type \char`\"{}.\+ttf\char`\"{} or \char`\"{}.\+ttc\char`\"{} fonts). True\+Type fonts are preferable over Base-\/14 fonts (which are P\+DF\textquotesingle{}s default fonts) since they provide partial (or complete) unicode support.

When using Base-\/14 fonts, \mbox{\hyperlink{classWt_1_1WString_a32f78326b7c2f213768645e04dfa790a}{W\+String\+::narrow()}} will be called on text which may result in loss of information. \mbox{\Hypertarget{classWt_1_1WPdfImage_a85dc5ebd08579d093c4253c7b2323b92}\label{classWt_1_1WPdfImage_a85dc5ebd08579d093c4253c7b2323b92}}
