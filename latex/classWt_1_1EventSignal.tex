\doxysubsection{Detailed Description}
A signal that conveys user-\/interface events. 

An Event\+Signal is a special Signal that may be triggered by user interface events such as a mouse click, key press, or focus change. They are made available through the library in widgets like \mbox{\hyperlink{classWt_1_1WInteractWidget}{W\+Interact\+Widget}}, and should not be instantiated directly.

In addition to the behaviour of Signal, they are capable of both executing client-\/side and server-\/side slot code. They may learn Java\+Script from C++ code, through stateless slot learning, when connected to a slot that has a stateless implementation, using \mbox{\hyperlink{classWt_1_1WObject_adaa163b9e92933f3b2ff4ec58e2734c6}{W\+Object\+::implement\+Stateless()}}. Or they may be connected to a \mbox{\hyperlink{classWt_1_1JSlot}{J\+Slot}} which provides manual Java\+Script code.

The typically relay UI event details, using event details objects like \mbox{\hyperlink{classWt_1_1WKeyEvent}{W\+Key\+Event}} or \mbox{\hyperlink{classWt_1_1WMouseEvent}{W\+Mouse\+Event}}.

Connects a slot that is specified as Java\+Script only. 

This will provide a client-\/side connection between the event and some Java\+Script code as implemented by the slot. Unlike other connects, this does not cause the event to propagated to the application, and thus the state changes caused by the Java\+Script slot are not tracked client-\/side.

The connection is tracked, taking into account the life-\/time of the \mbox{\hyperlink{classWt_1_1JSlot}{J\+Slot}} object, and can be updated by modifying the {\ttfamily slot}.
