\doxysubsection{Detailed Description}
A client-\/side Java\+Script matrix. 

A Java\+Script\+Matrix has methods that make it possible to do client-\/side calculations on matrices.

Using a \mbox{\hyperlink{classWt_1_1WGLWidget_1_1JavaScriptMatrix4x4}{Java\+Script\+Matrix4x4}}, GL parameters can be modified without communication with the server. The value of the \mbox{\hyperlink{classWt_1_1WGLWidget_1_1JavaScriptMatrix4x4}{Java\+Script\+Matrix4x4}} is updated server-\/side whenever an event is sent to the server.

Important\+: only the \mbox{\hyperlink{classWt_1_1WGLWidget_1_1JavaScriptMatrix4x4_a612ceb7e169d85b31aa1ae86a8b85d3c}{js\+Ref()}} of the return value from a call to W\+G\+L\+Widget\+::create\+Java\+Script\+Matrix() is a variable name that can be used in custom Java\+Script to modify a matrix from external scripts. The \mbox{\hyperlink{classWt_1_1WGLWidget_1_1JavaScriptMatrix4x4_a612ceb7e169d85b31aa1ae86a8b85d3c}{js\+Ref()}} of return values of operations refer to unnamed temporary objects -\/ rvalues in C++-\/lingo.

The \mbox{\hyperlink{classWt_1_1WGLWidget_1_1JavaScriptMatrix4x4}{Java\+Script\+Matrix4x4}} is represented in Java\+Script as an array of 16 elements. This array represents the values of the matrix in column-\/major order. It is either a Float32\+Array or a plain Java\+Script array.
