\doxysubsection{Detailed Description}
A widget that is painted using vector graphics. 

A painted widget is rendered from basic drawing primitives. Rendering is done not on the server but on the browser, using different rendering methods\+:

\tabulinesep=1mm
\begin{longtabu}spread 0pt [c]{*{3}{|X[-1]}|}
\hline
{\bfseries{Browser}}&{\bfseries{Methods}} &{\bfseries{Default method}} \\\cline{1-3}
Firefox 1.\+5+&Html\+Canvas, Inline\+S\+VG, Png\+Image &Html\+Canvas \\\cline{1-3}
Internet Explorer 6.\+0+&Inline\+V\+ML, Png\+Image &Inline\+V\+ML \\\cline{1-3}
Internet Explorer 9+&Html\+Canvas, Inline\+S\+VG, Png\+Image &Html\+Canvas \\\cline{1-3}
Safari&Html\+Canvas, Inline\+S\+VG, Png\+Image &Html\+Canvas \\\cline{1-3}
Opera&Inline\+S\+VG, Html\+Canvas$\ast$, Png\+Image &Inline\+S\+VG \\\cline{1-3}
other&?&Html\+Canvas, Png\+Image \\\cline{1-3}
\end{longtabu}


{\itshape $\ast$ Html\+Canvas occasionally suffers from rendering artefacts in Opera.}

The different rendering methods correspond to different \mbox{\hyperlink{classWt_1_1WPaintDevice}{W\+Paint\+Device}} implementations, from which this widget choses a suitable one depending on the browser capabilities and configuration.

If no Java\+Script is available, the Java\+Script-\/based Html\+Canvas will not be used, and Inline\+S\+VG will be used instead. The method used may be changed by using \mbox{\hyperlink{classWt_1_1WPaintedWidget_a18d1ee409a269274335d377fb17ba74d}{set\+Preferred\+Method()}}.

In some browsers, Inline\+S\+VG requires that the document is rendered as X\+H\+T\+ML. This must be enabled in the configuration file using the {\ttfamily $<$send-\/xhtml-\/mime-\/type$>$} option. By default, this option is off. Firefox 4 and Chrome do support svg in normal html mode.

The Png\+Image is the most portable rendering method, and may be the fastest if the painting is of high complexity and/or the image is fairly small.

To use a W\+Painted\+Widget, you must derive from it and reimplement \mbox{\hyperlink{classWt_1_1WPaintedWidget_ad8ce22eff41754c8616f45851f57fb1a}{paint\+Event(\+W\+Paint\+Device $\ast$paint\+Device)}}. To paint on a \mbox{\hyperlink{classWt_1_1WPaintDevice}{W\+Paint\+Device}}, you will need to use a \mbox{\hyperlink{classWt_1_1WPainter}{W\+Painter}}. Repainting is triggered by calling the \mbox{\hyperlink{classWt_1_1WPaintedWidget_ab4bcfb99b24bcc629be79750fa98f983}{update()}} method.

\doxyparagraph*{C\+SS}

Styling through C\+SS is not applicable.

\begin{DoxyNote}{Note}
A W\+Painted\+Widget requires that it is given a size using \mbox{\hyperlink{classWt_1_1WPaintedWidget_a4dc70782b6804671589e8f0c94e16e63}{resize()}} or by a layout manager.
\end{DoxyNote}
\doxyparagraph*{Client side interaction and repainting}

Resizes the widget. 

Specifies a fixed size for this widget, setting C\+SS {\ttfamily width} and {\ttfamily height} properties. By default a widget has automatic width and height, which sets a size for the widget following C\+SS rules.

When the widget is not managed by a layout manager, the automatic (natural) size of a widget depends on whether they widget is a {\itshape block} or {\itshape inline} widget\+:
\begin{DoxyItemize}
\item a {\itshape block} widget takes by default the width of the parent, and the height that it needs based on its contents
\item an {\itshape inline} widget takes the width and height that it needs based on its contents (possibly wrapping over multiple lines). The width and height of an inline widget cannot be changed (by the letter of C\+SS, although most browsers will react to it in varying ways).
\end{DoxyItemize}

When inserted in a layout manager, the size set will be used as a widget\textquotesingle{}s preferred size, but the widget may be given a different size by the layout manager based on available space and stretch factors. The actual size given by a layout manager may be retrieved by making the widget \char`\"{}layout size aware\char`\"{}, using \mbox{\hyperlink{classWt_1_1WWidget_a4f83592912a7f8fa4fd35dadde78ee74}{set\+Layout\+Size\+Aware()}}. If you have defined a {\ttfamily \char`\"{}wt\+Resize()\char`\"{}} Java\+Script method for the widget, then this method will also be called.

The default width and height of a widget is \mbox{\hyperlink{classWt_1_1WLength_a0cf39ca4225776879d56ade60320c31a}{W\+Length\+::\+Auto}}.
