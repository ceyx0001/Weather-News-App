\doxysubsection{Detailed Description}
A single node in a tree. 

A tree list is constructed by combining several tree node objects in a tree hierarchy, by passing the parent tree node as the last argument in the child node constructor, or by using \mbox{\hyperlink{classWt_1_1WTreeNode_aaf2263f185c0224ba65e011167a652eb}{add\+Child\+Node()}}, to add a child to its parent.

Each tree node has a label, and optionally a label icon pair. The icon pair offers the capability to show a different icon depending on the state of the node (expanded or collapsed). When the node has any children, a child count may be displayed next to the label using \mbox{\hyperlink{classWt_1_1WTreeNode_a5a625b5a94386386dabd36277d5f7c1b}{set\+Child\+Count\+Policy()}}.

Expanding a tree node it will collapse all its children, so that a user may collapse/expand a node as a short-\/cut to collapsing all children.

The treenode provides several policies to communicate the current contents of the tree to the client (if possible)\+: 
\begin{DoxyItemize}
\item Content\+Loading\+::\+Eager\+: the entire tree is transmitted to the client, and all tree navigation requires no further communication. 
\item Content\+Loading\+::\+Lazy\+: only the minimum is transmitted to the client. When expanding a node for the first time, only then it is transmitted to the client, and this may thus have some latency. 
\item Content\+Loading\+::\+Next\+Level\+: all leafs of visible children are transmitted, but not their children. This provides a good trade-\/off between bandwith use and interactivity, since expanding any tree node will happen instantly, and at the same time trigger some communication in the back-\/ground to load the next level of invisible nodes. 
\end{DoxyItemize}

The default policy is Content\+Loading\+::\+Lazy. Another load policy may be specified using \mbox{\hyperlink{classWt_1_1WTreeNode_ab01e87fa0fdaf8f8d0f23657cc5af9f3}{set\+Load\+Policy()}} on the root node and before adding any children. The load policy is inherited by all children in the tree.

There are a few scenarios where it makes sense to specialize the W\+Tree\+Node class. One scenario is create a tree that is populated dynamically while browsing. For this purpose you should reimplement the \mbox{\hyperlink{classWt_1_1WTreeNode_a4721df55d08218f4ca114a885d5018fb}{populate()}} method, whose default implementation does nothing. This method is called when \textquotesingle{}loading\textquotesingle{} the node. The exact moment for loading a treenode depends on the Load\+Policy.

A second scenario that is if you want to customize the look of the tree label (see \mbox{\hyperlink{classWt_1_1WTreeNode_ae9acd45dc1f8c6ce1cdde76503f9d9d1}{label\+Area()}}) or if you want to modify or augment the event collapse/expand event handling (see \mbox{\hyperlink{classWt_1_1WTreeNode_a11331560298bd530094b6bf21bb6e9e8}{do\+Expand()}} and \mbox{\hyperlink{classWt_1_1WTreeNode_a979c42ce96b60f787bac32803a986c23}{do\+Collapse()}}).

See \mbox{\hyperlink{classWt_1_1WTree}{W\+Tree}} for a usage example.
