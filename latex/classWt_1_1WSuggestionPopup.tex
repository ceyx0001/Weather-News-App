\doxysubsection{Detailed Description}
A widget which popups to assist in editing a textarea or lineedit. 

This widget may be associated with one or more \mbox{\hyperlink{classWt_1_1WFormWidget}{W\+Form\+Widgets}} (typically a \mbox{\hyperlink{classWt_1_1WLineEdit}{W\+Line\+Edit}} or a \mbox{\hyperlink{classWt_1_1WTextArea}{W\+Text\+Area}}).

The popup provides the user with suggestions to enter input. The popup can be used by one or more editors, using \mbox{\hyperlink{classWt_1_1WSuggestionPopup_a7be98d950273e379131aa1210798fd1a}{for\+Edit()}}. The popup will show when the user starts editing the edit field, or when the user opens the suggestions explicitly using a drop down icon or with the down key. The popup positions itself intelligently just below or just on top of the edit field. It offers a list of suggestions that match in some way with the current edit field, and dynamically adjusts this list. The implementation for matching individual suggestions with the current text is provided through a Java\+Script function. This function may also highlight part(s) of the suggestions to provide feed-\/back on how they match.

W\+Suggestion\+Popup is an M\+VC view class, using a simple \mbox{\hyperlink{classWt_1_1WStringListModel}{W\+String\+List\+Model}} by default. You can set a custom model using \mbox{\hyperlink{classWt_1_1WSuggestionPopup_ad8ddf026a9b11b488199f5225df4e9fa}{set\+Model()}}. The model can provide different text for the suggestion text (\mbox{\hyperlink{classWt_1_1ItemDataRole_ae3ecc48a21764c9fce36a9be37ccc2c1}{Wt\+::\+Item\+Data\+Role\+::\+Display}}) and value (\mbox{\hyperlink{classWt_1_1WSuggestionPopup_acba647adde23e76f2ce5446b60145aa0}{edit\+Role()}}). The member methods \mbox{\hyperlink{classWt_1_1WSuggestionPopup_a0beae292354fa79c2bbe68e9b11c8f01}{clear\+Suggestions()}} and \mbox{\hyperlink{classWt_1_1WSuggestionPopup_aaf9e6a5b8776533768f70a2ff44ab300}{add\+Suggestion()}} manipulate this model.

By default, the popup implements all filtering client-\/side. To support large datasets, you may enable server-\/side filtering of suggestions based on the input. The server-\/side filtering may provide a coarse filtering using a fixed size prefix of the entered text, and complement the client-\/side filtering. To enable server-\/side filtering, use \mbox{\hyperlink{classWt_1_1WSuggestionPopup_a8a77e3397fa98db0639fca3b1c94a8a2}{set\+Filter\+Length()}} and listen to filter notification using the model\+Filter() signal. Whenever a filter event is generated you can adjust the model\textquotesingle{}s content according to the filter (e.\+g. using a \mbox{\hyperlink{classWt_1_1WSortFilterProxyModel}{W\+Sort\+Filter\+Proxy\+Model}}). By using \mbox{\hyperlink{classWt_1_1WCompositeWidget_a8d6e43eec426afb7d96a200abe246ad7}{set\+Maximum\+Size()}} you can also limit the maximum height of the popup, in which case scrolling is supported (similar to a combo-\/box).

The class is initialized with an \mbox{\hyperlink{structWt_1_1WSuggestionPopup_1_1Options}{Options}} struct which configures how suggestion filtering and result editing is done. Alternatively, you can provide two Java\+Script functions, one for filtering the suggestions, and one for editing the value of the textarea when a suggestion is selected.
