\doxysubsection{Detailed Description}
A proxy model for Wt\textquotesingle{}s item models that provides filtering and/or sorting. 

This proxy model does not store data itself, but presents data from a source model, after filtering rows. It also allows sorting of the source model data, without actually altering the source model. This may be convenient when the source model does not support sorting (i.\+e. does not reimplement \mbox{\hyperlink{classWt_1_1WAbstractItemModel_ab044b92afa69282ea2e351bcdfd7c0c7}{W\+Abstract\+Proxy\+Model\+::sort()}}), or you do not want to reorder the underlying model since that affects all views on the model.

To use the proxy model to filter data, you use the methods \mbox{\hyperlink{classWt_1_1WSortFilterProxyModel_aaf272fe23196b67a409e7d0de03fd8d5}{set\+Filter\+Key\+Column()}}, \mbox{\hyperlink{classWt_1_1WSortFilterProxyModel_abd4e255015997a23017de5df48be3a8f}{set\+Filter\+Reg\+Exp()}} and \mbox{\hyperlink{classWt_1_1WSortFilterProxyModel_a813d0ddf54e2e7ce7ccbd1b2a2c6819f}{set\+Filter\+Role()}} to specify a filtering operation based on the values of a single column. If this filtering mechanism is too limiting, you can provide specialized filtering by reimplementing the \mbox{\hyperlink{classWt_1_1WSortFilterProxyModel_a9780b5aec4806a8f44a9fcf378b445a7}{filter\+Accept\+Row()}} method.

Sorting is provided by reimplementing the standard \mbox{\hyperlink{classWt_1_1WAbstractItemModel_ab044b92afa69282ea2e351bcdfd7c0c7}{W\+Abstract\+Item\+Model\+::sort()}} method. In this way, a view class such as \mbox{\hyperlink{classWt_1_1WTreeView}{W\+Tree\+View}} may resort the model as indicated by the user. Use \mbox{\hyperlink{classWt_1_1WSortFilterProxyModel_a097579ed002d4506771dc0eeb478ba4c}{set\+Sort\+Role()}} to indicate on what data role sorting should be done, or reimplement the \mbox{\hyperlink{classWt_1_1WSortFilterProxyModel_ad4f2c31c260dea967644e8378eed33da}{less\+Than()}} method to provide a specialized sorting method.

By default, the proxy does not automatically refilter and resort when the original model changes. \mbox{\hyperlink{classData}{Data}} changes or row additions to the source model are not automatically reflected in the proxy model, but to maintain integrity, row removals in the source model are always reflected in the proxy model. You can enable the model to always refilter and resort when the underlying model changes using \mbox{\hyperlink{classWt_1_1WSortFilterProxyModel_a4dfdc62d591249454cbb38e5099f19b4}{set\+Dynamic\+Sort\+Filter()}}.
