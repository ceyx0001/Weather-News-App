\doxysubsection{Detailed Description}
A validator is used to validate user input according to pre-\/defined rules. 

A validator may be associated with a form widget using \mbox{\hyperlink{classWt_1_1WFormWidget_a8b5fa8567e930691b95f1d0fa69359f1}{W\+Form\+Widget\+::set\+Validator()}}.

The validator validates the user input. A validator may have a split implementation to provide both validation at the client-\/side (which gives instant feed-\/back to the user while editing), and server-\/side validation (to be sure that the client was not tampered with). The feed-\/back given by (client-\/side and server-\/side) validation is reflected in the style class of the form field\+: a style class of {\ttfamily Wt-\/invalid} is set for a field that is invalid.

This W\+Validator only checks that mandatory fields are not empty. This class is reimplemented in \mbox{\hyperlink{classWt_1_1WDateValidator}{W\+Date\+Validator}}, \mbox{\hyperlink{classWt_1_1WIntValidator}{W\+Int\+Validator}}, \mbox{\hyperlink{classWt_1_1WDoubleValidator}{W\+Double\+Validator}}, \mbox{\hyperlink{classWt_1_1WLengthValidator}{W\+Length\+Validator}} and \mbox{\hyperlink{classWt_1_1WRegExpValidator}{W\+Reg\+Exp\+Validator}}. All these validators provide both client-\/side and server-\/side validation.

If these validators are not suitable, you can inherit from this class, and provide a suitable implementation to \mbox{\hyperlink{classWt_1_1WValidator_a8c3f8f39bfba125a161a8ff49cffd212}{validate()}} which does the server-\/side validation. If you want to provide client-\/side validation for your own validator, you may also reimplement \mbox{\hyperlink{classWt_1_1WValidator_a5067eb396def59bd0a94c33d69387c44}{java\+Script\+Validate()}}.
