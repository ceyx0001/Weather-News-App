\doxysubsection{Detailed Description}
A widget that renders an X\+H\+T\+ML template. 

The X\+H\+T\+ML template may contain references to variables which replaced by strings are widgets.

Since the template text may be supplied by a \mbox{\hyperlink{classWt_1_1WString}{W\+String}}, you can conveniently store the string in a message resource bundle, and make it localized by using \mbox{\hyperlink{classWt_1_1WString_adf1e7e5fc3c7199563a69bf07d77e571}{W\+String\+::tr()}}.

Placeholders (for variables and functions) are delimited by\+: {\ttfamily \$\{...\}}. To use a literal {\ttfamily \char`\"{}\$\{\char`\"{}}, use {\ttfamily \char`\"{}\$\$\{\char`\"{}}. Place holder names can contain \textquotesingle{}\+\_\+\textquotesingle{}, \textquotesingle{}-\/\textquotesingle{}, \textquotesingle{}.\textquotesingle{} and alfanumeric characters.

Usage example\+:

There are currently three syntactic constructs defined\+: variable place holders, functions and conditional blocks.

\doxyparagraph*{A. Variable placeholders}

{\ttfamily \$\{var\}} defines a placeholder for the variable \char`\"{}var\char`\"{}, and gets replaced with whatever is bound to that variable\+:
\begin{DoxyItemize}
\item a widget, using \mbox{\hyperlink{classWt_1_1WTemplate_a06b821ef6d6c684ea7c708a13d66fce7}{bind\+Widget()}}
\item a string value, using \mbox{\hyperlink{classWt_1_1WTemplate_a16e6f5aa8a866d95cb68a6ab8fbe753d}{bind\+String()}} or \mbox{\hyperlink{classWt_1_1WTemplate_a89426dcac3371befc127f7d5b04bb5ed}{bind\+Int()}}
\item or in general, the result of \mbox{\hyperlink{classWt_1_1WTemplate_a1b41ce6236c27e91bfffb2d0686ac556}{resolve\+String()}} and \mbox{\hyperlink{classWt_1_1WTemplate_a65fd50e2e83ef594b41a97b5a3f70588}{resolve\+Widget()}} methods.
\end{DoxyItemize}

Optionally, additional arguments can be specified using the following syntax\+:

{\ttfamily \$\{var arg1=\char`\"{}\+A value\char`\"{} arg2=\textquotesingle{}A second value\textquotesingle{}\}}

The arguments can thus be simple strings or quoted strings (single or double quoted). These arguments are applied to a resolved widget in \mbox{\hyperlink{classWt_1_1WTemplate_aa83a0d918e10eb3e8de6a747dcd6a3be}{apply\+Arguments()}} and currently supports only style classes.

You can bind widgets and values to variables using \mbox{\hyperlink{classWt_1_1WTemplate_a06b821ef6d6c684ea7c708a13d66fce7}{bind\+Widget()}}, \mbox{\hyperlink{classWt_1_1WTemplate_a16e6f5aa8a866d95cb68a6ab8fbe753d}{bind\+String()}} or \mbox{\hyperlink{classWt_1_1WTemplate_a89426dcac3371befc127f7d5b04bb5ed}{bind\+Int()}} or by reimplementing the \mbox{\hyperlink{classWt_1_1WTemplate_a1b41ce6236c27e91bfffb2d0686ac556}{resolve\+String()}} and \mbox{\hyperlink{classWt_1_1WTemplate_a65fd50e2e83ef594b41a97b5a3f70588}{resolve\+Widget()}} methods.

\begin{DoxyNote}{Note}
The use of X\+ML comments ({\ttfamily $<$!-\/-\/ ... -\/-\/$>$}) around variables that are bound to widgets will result in bad behaviour since the template parser is ignorant about these comments and the corresponding widgets will believe that they are rendered but aren\textquotesingle{}t actually.
\end{DoxyNote}
\doxyparagraph*{B. Functions}

{\ttfamily \$\{fun\+:arg\}} defines a placeholder for applying a function \char`\"{}fun\char`\"{} to an argument \char`\"{}arg\char`\"{}.

Optionally, additional arguments can be specified as with a variable placeholder.

\mbox{\hyperlink{structWt_1_1WTemplate_1_1Functions}{Functions}} are resolved by \mbox{\hyperlink{classWt_1_1WTemplate_a54a59255a73303197df91ead4a4b077a}{resolve\+Function()}}, and the default implementation considers functions bound with \mbox{\hyperlink{classWt_1_1WTemplate_ae8f4d8e21baa953323f3c0ce1cb4314a}{add\+Function()}}. There are currently three functions that are generally useful\+:
\begin{DoxyItemize}
\item \mbox{\hyperlink{}{\+: resolves a localized strings, this is convenient to create a language neutral template, which contains translated strings Functions\+::id}} \+: resolves the id of a bound widget, this is convenient to bind $<$label$>$ elements to a form widget using its for attribute.
\item \mbox{\hyperlink{}{\+: recursively renders another string as macro block optional arguments substituted before processing template substitution. For example, the following template uses the \char`\"{}tr\char`\"{} function to translate the age-\/label using the \char`\"{}age-\/label\char`\"{} internationalized key. }}C. Conditional blocks
\end{DoxyItemize}

{\ttfamily \$\{$<$cond$>$\}} starts a conditional block with a condition name \char`\"{}cond\char`\"{}, and must be closed by a balanced {\ttfamily \$\{$<$/cond$>$\}}.

For example\+:

Conditions are set using \mbox{\hyperlink{classWt_1_1WTemplate_ab2b6fc00ef28674be103dd1b13fd0aec}{set\+Condition()}}.

This widget does not provide styling, and can be styled using inline or external C\+SS as appropriate.

If a widget was already bound to the variable, it is deleted first. If previously a string or other value was bound to the variable, it is removed.

You may also pass a {\ttfamily nullptr} {\ttfamily widget}, which will resolve to an empty string.
