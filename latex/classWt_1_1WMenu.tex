\doxysubsection{Detailed Description}
A widget that shows a menu of options. 

The W\+Menu widget offers menu navigation.

Typically, a menu is used in conjunction with a \mbox{\hyperlink{classWt_1_1WStackedWidget}{W\+Stacked\+Widget}} (but can be used without too), where different \textquotesingle{}contents\textquotesingle{} are stacked upon each other. Each choice in the menu (which is implemented as a \mbox{\hyperlink{classWt_1_1WMenuItem}{W\+Menu\+Item}}) corresponds to a tab in the contents stack. The contents stack may contain other items, and could be shared with other \mbox{\hyperlink{classWt_1_1WMenu}{W\+Menu}} instances.

When used without a contents stack, you can react to menu item selection using the \mbox{\hyperlink{classWt_1_1WMenu_a0303879705bcdd8dbffc2d1528670d20}{item\+Selected()}} signal, to implement some custom handling of item selection.

Usage example\+:

After construction, the first entry will be selected. At any time, it is possible to select a particular item using \mbox{\hyperlink{classWt_1_1WMenu_a7bafc33564cb06d980d75e2f4d89f28d}{select()}}.

Each item of W\+Menu may be closeable (see \mbox{\hyperlink{classWt_1_1WMenuItem_a59b6dee6c95ae958f0c3b76d5ef7272d}{W\+Menu\+Item\+::set\+Closeable(bool)}}. Like selection, at any time, it is possible to close a particular item using \mbox{\hyperlink{classWt_1_1WMenu_a97c9be1df48cb0d27e2e4b904b51e184}{close()}}. You can react to close of item by using the \mbox{\hyperlink{classWt_1_1WMenu_ab9fa68e381972fb5385478b1e9eb6c78}{item\+Closed()}} signal.

The W\+Menu implementation offers fine-\/grained control on how contents should be preloaded. By default, all contents is lazy-\/loaded, only when needed. To improve response time, an item may also be preloaded (using \mbox{\hyperlink{classWt_1_1WMenu_ad0cc2914b7c0a5ee545013d8d7720e4d}{add\+Item()}}). In that case, the item will be loaded in the background, before its first use. In any case, once the contents corresponding to a menu item is loaded, subsequent navigation to it is handled entirely client-\/side.

The W\+Menu may participate in the application\textquotesingle{}s internal path, which lets menu items correspond to internal U\+R\+Ls, see \mbox{\hyperlink{classWt_1_1WMenu_ae1807eba0b1002808dbec779d00a79dc}{set\+Internal\+Path\+Enabled()}}.

The look of the items may be defined through style sheets. The default \mbox{\hyperlink{classWt_1_1WMenuItem}{W\+Menu\+Item}} implementation uses four style classes to distinguish between inactivated, activated, closeable inactivated and closeable activated menu items\+: {\ttfamily \char`\"{}item\char`\"{}}, {\ttfamily \char`\"{}itemselected\char`\"{}}, {\ttfamily \char`\"{}citem\char`\"{}}, {\ttfamily \char`\"{}citemselected\char`\"{}}. By using C\+SS nested selectors, a different style may be defined for items in a different menu.

You may customize the rendering and behaviour of menu entries by specializing \mbox{\hyperlink{classWt_1_1WMenuItem}{W\+Menu\+Item}}.

\doxyparagraph*{C\+SS}

The menu is rendered as a $<$ul$>$. Unless you use the bootstrap theme, you will want to customize the menu using inline or external styles to hide the bullets and provide the appropriate horizontal or vertical layout.

Loads content just before the widget is used. 

This function is called when a widget is inserted in the widget hierarchy. Widgets that get inserted in the widget hierarchy will be rendered. Visible widgets are rendered immediately, and invisible widgets in the back-\/ground (or not for a plain H\+T\+ML session). Enables internal paths for items. 

The menu participates in the internal path by changing the internal path when an item has been selected, and listening for path changes to react to path selections. As a consequence this allows the user to bookmark the current menu selection and revisit it later, use back/forward buttons to navigate through history of visited menu items, and allows indexing of pages.

For each menu item, \mbox{\hyperlink{classWt_1_1WMenuItem_afded7ede14b03c42cfec06f9de515e6d}{W\+Menu\+Item\+::path\+Component()}} is appended to the {\ttfamily base\+Path}, which defaults to the internal path (\mbox{\hyperlink{classWt_1_1WApplication_a43a0ed694ea2279e6cb33a8de4bc7536}{W\+Application\+::internal\+Path()}}). A \textquotesingle{}/\textquotesingle{} is appended to the base path, to turn it into a folder, if needed.

By default, menu interaction does not change the application internal path.
