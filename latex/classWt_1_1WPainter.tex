\doxysubsection{Detailed Description}
Vector graphics painting class. 

The painter class provides a vector graphics interface for painting. It needs to be used in conjunction with a \mbox{\hyperlink{classWt_1_1WPaintDevice}{W\+Paint\+Device}}, onto which it paints. To start painting on a device, either pass the device through the constructor, or use \mbox{\hyperlink{classWt_1_1WPainter_a6b81663b5599f0736ae5099ac7325d44}{begin()}}.

A typical use is to instantiate a W\+Painter from within a specialized \mbox{\hyperlink{classWt_1_1WPaintedWidget_ad8ce22eff41754c8616f45851f57fb1a}{W\+Painted\+Widget\+::paint\+Event()}} implementation, to paint on the given paint device, but you can also use a painter to paint directly to a particular paint device of choice, for example to create S\+VG, P\+DF or P\+NG images (as resources).

The painter maintains state such as the current \mbox{\hyperlink{classWt_1_1WPainter_a159ebf24df790e409d4060f7bd4aa725}{pen}}, \mbox{\hyperlink{classWt_1_1WPainter_ad63f01349e9163992d440e9aa85f916b}{brush}}, \mbox{\hyperlink{classWt_1_1WPainter_ad9cac6c8eaba94226f54343cde9676ea}{font}}, \mbox{\hyperlink{classWt_1_1WPainter_aaa5aed8b454e39316c4b78e0ffe57dff}{shadow}}, \mbox{\hyperlink{classWt_1_1WPainter_a0a509496b00579ec993d91850c44e05d}{transformation}} and clipping settings (see \mbox{\hyperlink{classWt_1_1WPainter_a4163898f999d4ecafec52c3f8890f380}{set\+Clipping()}} and \mbox{\hyperlink{classWt_1_1WPainter_a65be44d995c5f06f065f68d15c8889a5}{set\+Clip\+Path()}}). A particular state can be saved using \mbox{\hyperlink{classWt_1_1WPainter_a0262e162bb4be43f800c6c7b784c8d7a}{save()}} and later restored using \mbox{\hyperlink{classWt_1_1WPainter_a3fd8699e88d695cfd10043d18a666991}{restore()}}.

The painting system distinguishes between device coordinates, logical coordinates, and local coordinates. The device coordinate system ranges from (0, 0) in the top left corner of the device, to (device-\/$>$width().to\+Pixels(), device-\/$>$height().to\+Pixels()) for the bottom right corner. The logical coordinate system defines a coordinate system that may be chosen independent of the geometry of the device, which is convenient to make abstraction of the actual device size. Finally, the current local coordinate system may be different from the logical coordinate system because of a transformation set (using \mbox{\hyperlink{classWt_1_1WPainter_aaf5755eca15cb6e73fc24edf3bbd5758}{translate()}}, \mbox{\hyperlink{classWt_1_1WPainter_ab6d68d362e43f8e8df7e4400e2324d8e}{rotate()}}, and \mbox{\hyperlink{classWt_1_1WPainter_a0167a7c8c8d72ee0cca6f735b65a7938}{scale()}}). Initially, the local coordinate system coincides with the logical coordinate system, which coincides with the device coordinate system.

The device coordinates are defined in terms of pixels. Even though most underlying devices are actual vector graphics formats, when used in conjunction with a \mbox{\hyperlink{classWt_1_1WPaintedWidget}{W\+Painted\+Widget}}, these vector graphics are rendered by the browser onto a pixel-\/based canvas (like the rest of the user-\/interface). The coordinates are defined such that integer values correspond to an imaginary raster which separates the individual pixels, as in the figure below.



As a consequence, to avoid anti-\/aliasing effects when drawing straight lines of width one pixel, you will need to use vertices that indicate the middle of a pixel to get a crisp one-\/pixel wide line, as in the example figure.

By setting a \mbox{\hyperlink{classWt_1_1WPainter_a08c013cf3c07170319c48318bc586dc0}{view\+Port()}} and a \mbox{\hyperlink{classWt_1_1WPainter_a8d3d36b7be094fad53559c44239543dc}{window()}}, a view\+Port transformation is defined which maps logical coordinates onto device coordinates. By changing the world transformation (using \mbox{\hyperlink{classWt_1_1WPainter_ac0aa03db16f695b9bec3ef38588a5f3e}{set\+World\+Transform()}}, or \mbox{\hyperlink{classWt_1_1WPainter_aaf5755eca15cb6e73fc24edf3bbd5758}{translate()}}, \mbox{\hyperlink{classWt_1_1WPainter_ab6d68d362e43f8e8df7e4400e2324d8e}{rotate()}}, \mbox{\hyperlink{classWt_1_1WPainter_a0167a7c8c8d72ee0cca6f735b65a7938}{scale()}} operations), it is defined how current local coordinates map onto logical coordinates.

The painter provides support for clipping using an arbitrary \mbox{\hyperlink{classWt_1_1WPainterPath}{path}}, but not that the W\+Vml\+Image paint device only has limited support for clipping.

When rendering to an H\+T\+ML canvas, this will cause far less Java\+Script to be generated than separate calls to draw\+Path. Draws text. 

Draws text using inside the rectangle, using the current font. The text is aligned inside the rectangle following alignment indications given in {\ttfamily flags}. The text is drawn using the current transformation, pen color (\mbox{\hyperlink{classWt_1_1WPainter_a990bd8bef109cb8d97768b50975ca832}{pen()}}) and font settings (\mbox{\hyperlink{classWt_1_1WPainter_a4d4278fb25293129d93b20cdab059827}{font()}}).

Alignment\+Flags is the logical OR of a horizontal and vertical alignment. Orientation\+::\+Horizontal alignment may be one of Alignment\+Flag\+::\+Left, Alignment\+Flag\+::\+Center, or Alignment\+Flag\+::\+Right. Orientation\+::\+Vertical alignment is one of Alignment\+Flag\+::\+Top, Alignment\+Flag\+::\+Middle or Alignment\+Flag\+::\+Bottom.

Text\+Flag determines how the text is rendered in the rectangle. \mbox{\hyperlink{structText}{Text}} can be rendered on one line or by wrapping the words within the rectangle.

If a clip\+Point is provided, the text will not be drawn if the point is outside of the \mbox{\hyperlink{classWt_1_1WPainter_a255605d5f7c6425192cdd33ef64904b6}{clip\+Path()}}.

\begin{DoxyNote}{Note}
If the clip path is not a polygon, i.\+e. it has rounded segments in it, an approximation will be used instead by having the polygon pass through the control points, and the begin and end point of arcs.

Html\+Canvas\+: on older browsers implementing Html5 canvas, text will be rendered horizontally (unaffected by rotation and unaffected by the scaling component of the transformation matrix). In that case, text is overlayed on top of painted shapes (in D\+OM div\textquotesingle{}s), and is not covered by shapes that are painted after the text. Use the S\+VG and V\+ML renderers (W\+Painted\+Widget\+::inline\+Svg\+Vml) for the most accurate font rendering. Native H\+T\+M\+L5 text rendering is supported on Firefox3+, Chrome2+ and Safari4+.

Text\+Flag\+::\+Word\+Wrap\+: using the Text\+Flag\+::\+Word\+Wrap Text\+Flag is currently only supported by the S\+VG backend. The code generated by the S\+VG backend uses features currently only supported by Inkscape. Inkscape currently supports only Side\+::\+Top vertical alignments. 
\end{DoxyNote}
