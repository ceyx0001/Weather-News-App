\doxysubsection{Detailed Description}
A user control that represents a check box. 

By default, a checkbox can have two states\+: Wt\+::\+Check\+State\+::\+Checked or Wt\+::\+Check\+State\+::\+Unchecked, which can be inspected using \mbox{\hyperlink{classWt_1_1WAbstractToggleButton_a59099827039002a45133d4c7bad5310f}{is\+Checked()}}, and set using \mbox{\hyperlink{classWt_1_1WAbstractToggleButton_a549d62ab1ca78a3dfb65c19bedd79773}{set\+Checked()}}.

A checkbox may also provide a third state, Wt\+::\+Check\+State\+::\+Partially\+Checked, which is useful to indicate that it is neither checked nor unchecked. Wt will use native browser support for this H\+T\+M\+L5 extension when available (Safari and MS IE), and use an image-\/based workaround otherwise. You may enable support for the third state using \mbox{\hyperlink{classWt_1_1WCheckBox_aaa80606e36bfd34a73f6b58dc6cac926}{set\+Tristate()}}, and use \mbox{\hyperlink{classWt_1_1WCheckBox_a293d15f49d36587b55a04216322083d8}{set\+Check\+State()}} and \mbox{\hyperlink{classWt_1_1WCheckBox_ad006879a9fb12a6eb6dc52a63922505b}{check\+State()}} to read all three states. Once a tri-\/state checkbox is clicked, it cycles through the states Wt\+::\+Check\+State\+::\+Checked and Wt\+::\+Check\+State\+::\+Unchecked.

A label is added as a sibling of the checkbox to the same parent.

Usage example\+:

W\+Check\+Box is an \mbox{\hyperlink{classWt_1_1WWidget_ac78e3af143883334c82031790c87416e}{inline }} widget.

\doxyparagraph*{C\+SS}

This widget is rendered using an H\+T\+ML {\ttfamily $<$input type=\char`\"{}checkbox\char`\"{}$>$} tag. When a label is specified, the input element is nested in a {\ttfamily $<$label$>$}.

This widget does not provide styling, and can be styled using inline or external C\+SS as appropriate.
