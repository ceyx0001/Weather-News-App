\doxysubsection{Detailed Description}
An object which can be rendered in the H\+T\+TP protocol. 

\doxyparagraph*{Usage}

Besides the main page, other objects may be rendered as additional resources, for example documents or images. Classes such as \mbox{\hyperlink{classWt_1_1WAnchor}{W\+Anchor}} or \mbox{\hyperlink{classWt_1_1WImage}{W\+Image}} can use a resource instead of a U\+RL to provide their contents. Whenever the resource has changed, you should call the \mbox{\hyperlink{classWt_1_1WResource_a9394f6c2ff6cadf8d1076ec52c5e909a}{set\+Changed()}} method. \mbox{\hyperlink{classWt_1_1WResource_a9394f6c2ff6cadf8d1076ec52c5e909a}{set\+Changed()}} will make sure that the browser will use a new version of the resource by generating a new U\+RL, and emits the \mbox{\hyperlink{classWt_1_1WResource_a1527418354a584790c372e86a40c6027}{data\+Changed()}} signal to make those that refer to the resource aware that they should update their references to the new U\+RL.

You can help the browser to start a suitable helper application to handle the downloaded resource, or suggest to the user a suitable filename for saving the resource, by setting an appropriate file name using \mbox{\hyperlink{classWt_1_1WResource_a47632cad99e70712d4f898d97c3ea314}{suggest\+File\+Name()}}.

To serve resources that you create on the fly, you need to specialize this class and implement \mbox{\hyperlink{classWt_1_1WResource_a3a740f8cc29e2e6e4e58defc9c88b8b6}{handle\+Request()}}.

\doxyparagraph*{Concurrency issues}

Because of the nature of the web, a resource may be requested one time or multiple times at the discretion of the browser, and therefore your resource should in general not have any side-\/effects except for what is needed to render its own contents. Unlike event notifications to a Wt application, resource requests are not serialized, but are handled concurrently. You need to grab the application lock if you want to access or modify other widget state from within the resource. You can enable \mbox{\hyperlink{classWt_1_1WResource_af0b52160a8d04700a17e9479dd489972}{takes\+Update\+Lock()}} to let W\+Resource do that for you. When deleting a resource, any pending request is cancelled first. For this mechanism to work you need to specialize the destructor and call \mbox{\hyperlink{classWt_1_1WResource_a24f65859427f54012fef2fa9da14870d}{being\+Deleted()}}. This method may safely be called multiple times (i.\+e. from within each destructor in the hierachy).

\doxyparagraph*{Continuations for asynchronous I/O}

\mbox{\hyperlink{structWith}{With}} respect to I/O, the current strategy is to cache the whole response first in a buffer and use async I/O to push the data to the client, in order to free the thread while waiting for I/O on a possibly slow link. You do not necessarily have to provide all output at once, instead you can obtain a \mbox{\hyperlink{classWt_1_1Http_1_1ResponseContinuation}{Http\+::\+Response\+Continuation}} object for a response, construct the response piecewise. A new request() will be made to continue the response.

\doxyparagraph*{Global and private resources}

By default, a resource is private to an application\+: access to it is protected by same secret session Id that protects any other access to the application.

You can also deploy static resources, which are global, using \mbox{\hyperlink{classWt_1_1WServer_a3c5c323a9bccf591b63c9fb3f40e8b59}{W\+Server\+::add\+Resource()}}.

\doxyparagraph*{Monitoring upload progress}

A resource may also handle the uploading of files (in fact, \mbox{\hyperlink{classWt_1_1WFileUpload}{W\+File\+Upload}} uses a \mbox{\hyperlink{classWt_1_1WResource}{W\+Resource}} to do exactly that) or transmission of other large bodies of data being P\+O\+ST\textquotesingle{}ed or P\+UT to the resource U\+RL. For these requests, it may be convenient to enable upload progress monitoring using \mbox{\hyperlink{classWt_1_1WResource_ae0b96cdd80b982c74adb9a762c77d479}{set\+Upload\+Progress()}}, which allows you to be notified of data being received.

By default, W\+Resource does not keep the \mbox{\hyperlink{classWt_1_1WApplication}{W\+Application}}\textquotesingle{}s update lock, so \mbox{\hyperlink{classWt_1_1WResource_a3a740f8cc29e2e6e4e58defc9c88b8b6}{handle\+Request()}} is done outside of the W\+Application\textquotesingle{}s event loop.
