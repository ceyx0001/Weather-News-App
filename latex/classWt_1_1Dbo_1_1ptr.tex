\doxysubsection{Detailed Description}
A smart pointer for a database object. 

This smart pointer class implements a reference counted shared pointer for database objects, which also keeps tracking of synchronization between the in-\/memory copy and the database copy. You should always use this pointer class to reference a database object.

Unlike typical C++ data structures, classes mapped to database tables do not have clear ownership relationships. Therefore, the conventional ownership-\/based memory allocation/deallocation does not work naturally for database classes.

A pointer may point to a {\itshape transient} object or a {\itshape persisted} object. A persisted object has a corresponding copy in the database while a transient object is only present in memory. To persist a new object, use \mbox{\hyperlink{classWt_1_1Dbo_1_1Session_ad762cc052a1bdc1ef42d6b90bcf1ee1b}{Session\+::add()}}. To make a persisted object transient, use \mbox{\hyperlink{classWt_1_1Dbo_1_1ptr_af959d9c3b8c5ccb2ab29728c3698f343}{remove()}}.

Unlike a typical smart pointer, this pointer only allows read access to the underlying object by default. To modify the object, you should explicitly use \mbox{\hyperlink{classWt_1_1Dbo_1_1ptr_aed605599617e3ae02d9e92b1daafa8e7}{modify()}}. This is used to mark the underyling object as {\itshape dirty} to add it to the queue of objects to be synchronized with the database.

The pointer class provides a number of methods to deal with the persistence state of the object\+:
\begin{DoxyItemize}
\item \mbox{\hyperlink{classWt_1_1Dbo_1_1ptr_a43ea33191c46985752c192edb8254331}{id()}}\+: returns the database id
\item \mbox{\hyperlink{classWt_1_1Dbo_1_1ptr_a3e4e12079f2e3595bc201207217b6a4f}{flush()}}\+: forces the object to be synchronized to the database
\item \mbox{\hyperlink{classWt_1_1Dbo_1_1ptr_af959d9c3b8c5ccb2ab29728c3698f343}{remove()}}\+: deletes the object in the underlying database
\item \mbox{\hyperlink{classWt_1_1Dbo_1_1ptr_aa8bfb5e7230ce4298f858d82022ec4e4}{reread()}}\+: rereads the database copy of the object
\item \mbox{\hyperlink{classWt_1_1Dbo_1_1ptr_a4018f9d4f4f15c485023968b0c73c3ea}{purge()}}\+: purges the transient version of a non-\/dirty object.
\end{DoxyItemize}

Wt\+::\+Dbo\+::ptr$<$const C$>$ can be used when retrieving query results. There are overloads for the copy constructor, copy assignment, and comparison operators to make this work as expected.
