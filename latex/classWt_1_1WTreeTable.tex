\doxysubsection{Detailed Description}
A table with a navigatable tree in the first column. 

A W\+Tree\+Table implements a tree table, where additional data associated is associated with tree items, which are organized in columns.

Unlike the M\+V\+C-\/based \mbox{\hyperlink{classWt_1_1WTreeView}{W\+Tree\+View}} widget, the tree renders a widget hierarchy, rather than a hierarhical standard model. This provides extra flexibility (as any widget can be used as contents), at the cost of server-\/side, client-\/side and bandwidth resources (especially for large tree tables).

The actual data is organized and provided by \mbox{\hyperlink{classWt_1_1WTreeTableNode}{W\+Tree\+Table\+Node}} widgets.

To use the tree table, you need {\bfseries{first}} to call \mbox{\hyperlink{classWt_1_1WTreeTable_a2ec7655241ad65f99c114d6ed37b49a1}{add\+Column()}} to specify the additional data columns. Next, you need to set the tree root using \mbox{\hyperlink{classWt_1_1WTreeTable_afa1813751aa1c4df798f4a5207e16547}{set\+Tree\+Root()}} and bind additional information (text or other widgets) in each node using \mbox{\hyperlink{classWt_1_1WTreeTableNode_a85d9b06b7329c0ae9b951c5680a02c6c}{W\+Tree\+Table\+Node\+::set\+Column\+Widget()}}. Thus, you cannot change the number of columns once the tree root has been set.

The table cannot be given a height using C\+SS style rules, instead you must use layout managers, or use \mbox{\hyperlink{classWt_1_1WCompositeWidget_a95584ea20561ba00e8c58156a24ba238}{resize()}}.
