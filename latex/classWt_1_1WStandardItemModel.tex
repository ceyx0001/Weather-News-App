\doxysubsection{Detailed Description}
A standard data model, which stores its data in memory. 

The standard item model supports all features of \mbox{\hyperlink{classWt_1_1WAbstractItemModel}{W\+Abstract\+Item\+Model}}, and can thus be used to represent tables, trees and tree tables.

The data itself are organized in \mbox{\hyperlink{classWt_1_1WStandardItem}{W\+Standard\+Item}} objects. There is one invisible root object (\mbox{\hyperlink{classWt_1_1WStandardItemModel_a59ab3e7ed361d7b443dcde92ecef1302}{invisible\+Root\+Item()}}) that holds the toplevel data. Most methods in this class that access or manipulate data internally operate on this root item.

If you want to use the model as a table, then you can use \mbox{\hyperlink{classWt_1_1WStandardItemModel_aa98fbf4d2acba75ea7a550f312fd0aef}{W\+Standard\+Item\+Model(int, int)}} to set the initial table size, and use the \mbox{\hyperlink{classWt_1_1WStandardItemModel_a8e14be9cf99523a3bb83185fc1c5a910}{item()}} and \mbox{\hyperlink{classWt_1_1WStandardItemModel_a19e4b7225467e7e674e7afc74e423fe7}{set\+Item()}} methods to set data. You can change the geometry by inserting rows (\mbox{\hyperlink{classWt_1_1WStandardItemModel_a6ac5a019999e2ac86d11801055ae36a5}{insert\+Row()}}) or columns (\mbox{\hyperlink{classWt_1_1WStandardItemModel_a30553370b2c3208c1bd637a2fe78fc9f}{insert\+Column()}}) or removing rows (\mbox{\hyperlink{classWt_1_1WAbstractItemModel_ada64c5f4d4eba6cecab2c1e443372cb3}{remove\+Row()}}) or columns (\mbox{\hyperlink{classWt_1_1WAbstractItemModel_a7b46c63a32eb9ca9838a1a7cd5853552}{remove\+Column()}}).

If you want to use the model as a tree (or tree table), then you can use the default constructor to start with an empty tree, and use the \mbox{\hyperlink{classWt_1_1WStandardItem}{W\+Standard\+Item}} A\+PI on \mbox{\hyperlink{classWt_1_1WStandardItemModel_a59ab3e7ed361d7b443dcde92ecef1302}{invisible\+Root\+Item()}} to manipulate the tree root. When you are building a tree, the column count at each node is 1. When you are building a tree table, you can add additional columns of data for each internal node. Only the items in the first column have children that result in a hierarchical tree structure.

When using the model with a view class, you can use the \mbox{\hyperlink{classWt_1_1WStandardItemModel_a4b29f87e5a3bc91776aefcd538be25be}{item\+From\+Index()}} and \mbox{\hyperlink{classWt_1_1WStandardItemModel_a9d0a7bdd426a693404997c05b9d6ed6d}{index\+From\+Item()}} models to translate between model indexes (that are used by the view class) and standard items.

Handles a drop event. 

The default implementation only handles generic drag\&drop between abstract item models. Source item data is copied (but not the source item\textquotesingle{}s flags).

This method is overloaded for handling drop events on top of items or drop events between items (see \mbox{\hyperlink{classWt_a46e584f830f2d8c9f5b16fcee7affc9b}{Wt\+::\+Drop\+Location}}). This overload handles drops on top of items, but note that due to historical reasons it will also insert the items in between when called with Drop\+Action\+::\+Move.

The location in the model is indicated by the {\ttfamily row} and {\ttfamily column} within the {\ttfamily parent} index. If {\ttfamily row} is -\/1, then the item is appended to the {\ttfamily parent}. Otherwise, the item is inserted at or copied over the indicated item (and subsequent rows). When {\ttfamily action} is a \mbox{\hyperlink{}{Drop\+Action\+::\+Move}}, the original items are deleted from the source model.

You may want to reimplement this method if you want to handle other mime-\/type data, or if you want to refine how the drop event of an item selection must be interpreted.

\begin{DoxyNote}{Note}
Currently, only row selections are handled by the default implementation.
