The recommended way to use any of these libraries besides {\ttfamily \char`\"{}tz.\+h\char`\"{}} is to just include it. These are header-\/only libraries (except {\ttfamily \char`\"{}tz.\+h\char`\"{}}).

To use {\ttfamily \char`\"{}tz.\+h\char`\"{}}, there is a single source file ({\ttfamily src/tz.\+cpp}) that needs to be compiled. Here are the recommended directions\+: \href{https://howardhinnant.github.io/date/tz.html\#Installation}{\texttt{ https\+://howardhinnant.\+github.\+io/date/tz.\+html\#\+Installation}}.

One can run tests by cd\textquotesingle{}ing into the {\ttfamily test} subdirectory and running {\ttfamily testit}. There are known failures on all platforms except for mac\+OS. And even on mac\+OS if C++11 is used. If any of these failures present problems for you, there exist workarounds.

Additionally there is {\itshape unsupported} support for \href{https://github.com/Microsoft/vcpkg}{\texttt{ vcpkg}} and \href{https://cmake.org/}{\texttt{ C\+Make}}. I don\textquotesingle{}t personally use or maintain these systems as for me they cause more problems than they solve (for this small project). If you would like to contribute to these build systems please feel free to file a PR.
