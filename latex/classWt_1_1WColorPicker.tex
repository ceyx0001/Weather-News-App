\doxysubsection{Detailed Description}
A widget that provides a browser-\/native color picker. 

To act upon color value changes, connect a slot to the \mbox{\hyperlink{classWt_1_1WFormWidget_a880589c79f1b93e7f38330e28cf6f5e2}{changed()}} signal. This signal is emitted when the user changes the selected color, and subsequently closes the color picker.

To act upon any color change, connect a slot to the \mbox{\hyperlink{classWt_1_1WColorPicker_a935fe2494566aa8b38092cac73c30b3c}{color\+Input()}} signal. \mbox{\hyperlink{classNote}{Note}} that this signal may fire very quickly depending on how the browser\textquotesingle{}s color picker works.

At all times, the currently selected color may be accessed with the value() method.

The widget corresponds to the H\+T\+ML {\ttfamily $<$input type=\char`\"{}color\char`\"{}$>$} tag. \mbox{\hyperlink{classNote}{Note}} that this element does not support C\+SS color names. When manipulating this widget with \mbox{\hyperlink{classWt_1_1WColor}{W\+Color}} values, ensure they have valid R\+GB values or the color picker will reset to \#000000.

W\+Color\+Picker is an \mbox{\hyperlink{classWt_1_1WWidget_ac78e3af143883334c82031790c87416e}{inline }} widget.

Event signal emitted when the selected color is changed. 

This signal is emitted whenever the selected color has changed. Unlike the \mbox{\hyperlink{classWt_1_1WFormWidget_a880589c79f1b93e7f38330e28cf6f5e2}{changed()}} signal, this signal is fired on every change, not only when the color picker is closed.

In particular, on browsers with a draggable color picker (i.\+e. most common browsers), this signal fires every time the position changes. Use with caution.
