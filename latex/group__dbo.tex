Maps the \char`\"{}\+One\char`\"{}-\/side (foreign key) of a Many\+To\+One or One\+To\+One relation. 

This function binds the pointer field {\ttfamily value} to the database foreign key field(s) {\ttfamily name} {\ttfamily + \char`\"{}\+\_\+\char`\"{} +} (C\textquotesingle{}s primary key(s)).

If the name starts with a {\ttfamily $>$}, the {\ttfamily $>$} is omitted and the name is used literally, instead of {\ttfamily name} {\ttfamily + \char`\"{}\+\_\+\char`\"{} +} (C\textquotesingle{}s primary key(s)), e.\+g. if the primary key of C is {\ttfamily id}, then the name {\ttfamily foo} translates to a column with the name {\ttfamily foo\+\_\+id}, and the name {\ttfamily $>$foo} translates to a column with the name {\ttfamily foo}.

If the name is omitted or empty, then C\textquotesingle{}s mapped table name is used.

A \mbox{\hyperlink{group__dbo_gac089020eff000d9e2c4ef72cf715bd21}{belongs\+To()}} will usually have a counter-\/part \mbox{\hyperlink{group__dbo_ga570b71f937c991b000b9db01b327a363}{has\+Many()}} or \mbox{\hyperlink{group__dbo_ga00a060d5f4764f673948989f21837548}{has\+One()}} declaration in the referenced class {\ttfamily C}.
