\doxysubsection{Detailed Description}
A utility class which provides timer signals and single-\/shot timers. 

To use a timer, create a W\+Timer instance, set the timer interval using \mbox{\hyperlink{classWt_1_1WTimer_ab1936d0a142b1ee97f2ee759f55c04f3}{set\+Interval()}} and connect a slot to the timeout signal. Then, start the timer using \mbox{\hyperlink{classWt_1_1WTimer_ae167fbd3eb981a2fce52e9a4b3b6e0af}{start()}}. An active timer may be cancelled at any time using \mbox{\hyperlink{classWt_1_1WTimer_a73c5d1b882af4835384ccdc07a650e34}{stop()}}.

By default, a timer will continue to generate events until you \mbox{\hyperlink{classWt_1_1WTimer_a73c5d1b882af4835384ccdc07a650e34}{stop()}} it. To create a timer that will fire only once, use \mbox{\hyperlink{classWt_1_1WTimer_a92ae68d15de600174afdd2e443397604}{set\+Single\+Shot()}}.

When connecting stateless slot implementations to the timeout signal, these stateless slot implementations will be used as for any other signal (when Ajax is available).

In clients without (enabled) Java\+Script support, the minimum resolution of the timer is one second (1000 milli-\/seconds), and it is probably wise to use timers sparingly.

A \mbox{\hyperlink{classWt_1_1WTimer}{W\+Timer}} is only usable inside of a Wt event loop. If you want to create a timer outside the Wt event loop, take a look at \{javadoclink java.\+util.\+Timer\}.
