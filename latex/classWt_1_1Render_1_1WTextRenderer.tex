\doxysubsection{Detailed Description}
An X\+H\+T\+ML renderering engine. 

This class implements a rendering engine for a (subset of) X\+H\+T\+M\+L/\+C\+SS. Its intended use is to be able to accurately render the output of the \mbox{\hyperlink{classWt_1_1WTextEdit}{W\+Text\+Edit}} widget (although it handles a more general subset of X\+H\+T\+M\+L/\+C\+SS than is required to do just that). Its focus is on high-\/quality rendering of text-\/like contents.

The following are the main features\+:
\begin{DoxyItemize}
\item decent rendering of inline (text) contents, floats, tables, images, ordered lists, unordered lists, in any arbitrary combination, with mixtures of font sizes, etc...
\item C\+SS stylesheet support using in-\/document $<$style$>$, inline style attributes or using \mbox{\hyperlink{classWt_1_1Render_1_1WTextRenderer_a8d915e0103401786b41543dfcedec662}{set\+Style\+Sheet\+Text()}}.
\item support for relative and absolute positioned layout contexts
\item support for (true type) fonts, font styles, font sizes and text decorations
\item support for text alignment options, padding, margins, borders, background color (only on block elements) and text colors
\item supports automatic and C\+SS page breaks (page-\/break-\/after or page-\/break-\/before)
\item supports font scaling (rendering text at another D\+PI than the rest)
\item can be used in conjunction with other drawing instructions to the same paint device.
\end{DoxyItemize}

Some of the main limitations are\+:
\begin{DoxyItemize}
\item only \char`\"{}display\+: inline\char`\"{} or \char`\"{}display\+: block\char`\"{} elements are supported. \char`\"{}display\+: none\char`\"{} and \char`\"{}display\+: inline-\/block\char`\"{} are not (yet) recognized.
\item only colors defined in terms of R\+GB values are supported\+: C\+SS named colors (e.\+g. \textquotesingle{}blue\textquotesingle{}) are not allowed.
\item Bidi (Right-\/to-\/\+Left) text rendering is not supported
\end{DoxyItemize}

The renderer is not C\+SS compliant (simply because it is still lacking alot of features), but the subset of C\+SS that is supported is a pragmatic choice. If things are lacking, let us known in the bug tracker.

This class is an abstract class. A concrete class implements the pure virtual methods to create an appropriate \mbox{\hyperlink{classWt_1_1WPaintDevice}{W\+Paint\+Device}} for each page and to provide page dimension information.

All coordinates and dimensions in the A\+PI below are pixel coordinates.

Renders an X\+H\+T\+ML fragment. 

The text is rendered, starting at position {\ttfamily y}, and flowing down the page. New pages are created using {\ttfamily \mbox{\hyperlink{classWt_1_1Render_1_1WTextRenderer_a9b9230eb8210d5bb2408321ebc88c07d}{start\+Page()}}} to render more contents on a next page. The return value is the position at which rendering stopped on the last page on which was rendered.

This {\ttfamily y} position and returned position are {\itshape text coordinates}, which differ from page coordinates in that they exclude margins.

The function returns the end position. You may call this function multiple times.

Each invocation to \mbox{\hyperlink{classWt_1_1Render_1_1WTextRenderer_ada84c9867c07a60c54e3f51e4b6426f5}{render()}} has the effect of resetting the logical page numbering used by \mbox{\hyperlink{classWt_1_1Render_1_1WTextRenderer_a49c53f92c778a63da84d8fd8dc0d876d}{page\+Width()}}, \mbox{\hyperlink{classWt_1_1Render_1_1WTextRenderer_a86a2e5a8042e5ea8d50593597e332791}{page\+Height()}} and \mbox{\hyperlink{classWt_1_1Render_1_1WTextRenderer_a9b9230eb8210d5bb2408321ebc88c07d}{start\+Page()}} so that the current page is page 0. \mbox{\Hypertarget{classWt_1_1Render_1_1WTextRenderer_a2ca18a73ebbe013b6913a277408eb2a6}\label{classWt_1_1Render_1_1WTextRenderer_a2ca18a73ebbe013b6913a277408eb2a6}}

Sets the contents of a cascading style sheet (C\+SS).
