\doxysubsection{Detailed Description}
A simple logging class. 

This class logs events to a stream in a flexible way. It allows to create log files using the commonly used \href{http://www.w3.org/Daemon/User/Config/Logging.html\#common-logfile-format}{\texttt{ Common Log Format}} or \href{http://httpd.apache.org/docs/1.3/logs.html\#combined}{\texttt{ Combined Log Format}}, but provides a general way for logging entries that consists of a fixed number of fields.

It is used by Wt to create the application log (\mbox{\hyperlink{classWt_1_1WApplication_aa806c98262a4d5c4d4d0537da59cdf1d}{W\+Application\+::log()}}), and built-\/in httpd access log.

To use this class for custom logging, you should instantiate a logger, add one or more field definitions using \mbox{\hyperlink{classWt_1_1WLogger_a8287b40a0699797f63de2fe5deaaae9a}{add\+Field()}}, and set an output stream using \mbox{\hyperlink{classWt_1_1WLogger_ae851860f76c299bca96321b7d867a503}{set\+Stream()}} or \mbox{\hyperlink{classWt_1_1WLogger_a6d8a07bc92824fe32a123094e7516544}{set\+File()}}. To stream data to the logger, use \mbox{\hyperlink{classWt_1_1WLogger_ad807be272b8acd4d01f5be3a601386b6}{entry()}} to start formatting a new entry.

{\footnotesize\ttfamily void Wt\+::\+W\+Logger\+::configure (\begin{DoxyParamCaption}\item[{const std\+::string \&}]{config }\end{DoxyParamCaption})}



Configures what things are logged. 

The configuration is a string that defines rules for enabling or disabling certain logging. It is a white-\/space delimited list of rules, and each rule is of the form\+:


\begin{DoxyItemize}
\item {\ttfamily \mbox{[}-\/\mbox{]}level} \+: enables (or disables) logging of messages of the given level; \textquotesingle{}$\ast$\textquotesingle{} is a wild-\/card that matches all levels
\item {\ttfamily \mbox{[}-\/\mbox{]}level\+:scope} \+: enables (or disables) logging of messages of the given level and scope; \textquotesingle{}$\ast$\textquotesingle{} is a wild-\/card that matches all levels or scopes.
\end{DoxyItemize}

The default configuration is {\ttfamily \char`\"{}$\ast$ -\/debug\char`\"{}}, i.\+e. by default everything is logged, except for \char`\"{}debug\char`\"{} messages.

Some other examples\+:


\begin{DoxyItemize}
\item {\ttfamily \char`\"{}$\ast$ -\/debug debug\+:wthttp\char`\"{}}\+: logs everything, including debugging messages of scope \char`\"{}wthttp\char`\"{}, but no other debugging messages.
\item {\ttfamily \char`\"{}$\ast$ -\/info -\/debug\char`\"{}}\+: disables logging of info messages in addition to debugging messages.
\end{DoxyItemize}

\begin{DoxyNote}{Note}
The standard logging is typically configured in the configuration file, in the $<$log-\/config$>$ block. 
\end{DoxyNote}

{\footnotesize\ttfamily void Wt\+::\+W\+Logger\+::set\+File (\begin{DoxyParamCaption}\item[{const std\+::string \&}]{path }\end{DoxyParamCaption})}



Sets the output file. 

Opens a file output stream for {\ttfamily path}. The default logger outputs to stderr.

This logs a message notifying the user whether the file was successfully opened for writing to the previous ostream (usually std\+::cerr).

If you want to suppress the info message, you can configure the logger with \char`\"{}-\/info\+:\+W\+Logger\char`\"{}. The error when the file was not successfully opened is logged at the error log level.

\begin{DoxyNote}{Note}
If the previous ostream was a file set with this method, the message will be logged to std\+::cerr instead of the previous file, because the previous file is closed first.
\end{DoxyNote}
