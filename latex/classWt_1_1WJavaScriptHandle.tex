\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename T$>$\newline
class Wt\+::\+W\+Java\+Script\+Handle$<$ T $>$}

A handle to a Java\+Script representation of an object. 

A W\+Java\+Script\+Handle allows to access and modify an object in Java\+Script. This is useful to avoid server roundtrips when frequently updating something, e.\+g. to interact with and animate a \mbox{\hyperlink{classWt_1_1WPaintedWidget}{W\+Painted\+Widget}}.

You can use the \mbox{\hyperlink{classWt_1_1WJavaScriptHandle_a8084d9b7795f2e27ee71b18fccb2f5d7}{value()}} of a W\+Java\+Script\+Handle just as you would normally, with the exception that it will be \mbox{\hyperlink{classWt_1_1WJavaScriptExposableObject_a4e121ccbb14715c33dc1b019b6636156}{Java\+Script bound}}, and so will any copies you make of it. You should not modify a Java\+Script bound object, as this will not change its client side representation. Use the handle\textquotesingle{}s \mbox{\hyperlink{classWt_1_1WJavaScriptHandle_ac1e446a7ac75a19e14d1019b8011499b}{set\+Value()}} method instead.

You can access (and modify) the value of a handle on the client side using \mbox{\hyperlink{classWt_1_1WJavaScriptHandle_a0d0bc00f0bcafb596c3f6c96c4241ded}{js\+Ref()}}.

You can update the value from the server with \mbox{\hyperlink{classWt_1_1WJavaScriptHandle_ac1e446a7ac75a19e14d1019b8011499b}{set\+Value()}}. Changes on the client side will be synced back to the server.

Currently, only \mbox{\hyperlink{classWt_1_1WPaintedWidget}{W\+Painted\+Widget}} allows the use of \mbox{\hyperlink{classWt_1_1WJavaScriptExposableObject}{Java\+Script exposable objects}}.
