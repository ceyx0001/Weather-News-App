\doxysubsection{Detailed Description}
Model for implementing an authentication view. 

This model implements the logic for authenticating a user (the \char`\"{}login\char`\"{} interface). It implements traditional username/password registration, and third party identification methods (although for the latter, it doesn\textquotesingle{}t really do anything).

The model exposes three fields\+:
\begin{DoxyItemize}
\item Login\+Name\+Field\+: the login name (used as an identity for the \mbox{\hyperlink{classWt_1_1Auth_1_1Identity_a1d680fd0dbf7a12684a511d74081eb06}{Identity\+::\+Login\+Name}} provider)
\item Password\+Field\+: the password
\item Remember\+Me\+Field\+: whether the login should be remembered with an authentication cookie (if that is configured in the \mbox{\hyperlink{classWt_1_1Auth_1_1AuthService}{Auth\+Service}}).
\end{DoxyItemize}

When the model validates correctly (\mbox{\hyperlink{classWt_1_1Auth_1_1AuthModel_a27faf850dfa6204b11719af1451a49c2}{validate()}} returns {\ttfamily true}), the entered credentials are correct. At that point you can use the \mbox{\hyperlink{classWt_1_1Auth_1_1AuthModel_a45e58ea275b2f313703948cf42f5a61b}{login()}} utility function to login the identified user.

The model can also be used when the user is already known (e.\+g. to implement password confirmation before a critical operation). In that case you can set a value for the Login\+Name\+Field and make this field invisible or read-\/only.

The model also provides the client-\/side Java\+Script logic to indicate password attempt throttling (\mbox{\hyperlink{classWt_1_1Auth_1_1AuthModel_a278e9ea2fd31530d677b7d6c331fdc95}{configure\+Throttling()}} and \mbox{\hyperlink{classWt_1_1Auth_1_1AuthModel_acd2acd450a3acdb984d9215b6e599c96}{update\+Throttling()}}).
