\doxysubsection{Detailed Description}
An abstract base class for radio buttons and check boxes. 

A toggle button provides a button with a boolean state (checked or unchecked), and a text label.

To act on a change of the state, either connect a slot to the \mbox{\hyperlink{classWt_1_1WFormWidget_a880589c79f1b93e7f38330e28cf6f5e2}{changed()}} signal, or connect a slot to the \mbox{\hyperlink{classWt_1_1WAbstractToggleButton_af80f466ccc3e8c3e19fb378879e16784}{checked()}} or \mbox{\hyperlink{classWt_1_1WAbstractToggleButton_a21b2c473366b8225202d9e2902b6a3c7}{un\+Checked()}} signals.

The current state (checked or unchecked) may be inspected using the \mbox{\hyperlink{classWt_1_1WAbstractToggleButton_a59099827039002a45133d4c7bad5310f}{is\+Checked()}} method.

This signal is not emitted for \mbox{\hyperlink{classWt_1_1WRadioButton}{W\+Radio\+Button}}. In this case a button can only be unchecked because another button is checked. The browser will only generate an event for the button which was checked. Since \mbox{\hyperlink{classWt_1_1EventSignal}{Event\+Signal}} mirrors browser events, \mbox{\hyperlink{classWt}{Wt}} will also not emit this signal. The function \mbox{\hyperlink{classWt_1_1WButtonGroup_aab57ae1e05bad3cf8d10e57e3067de75}{W\+Button\+Group\+::checked\+Changed()}} might be easier to use in this case.
