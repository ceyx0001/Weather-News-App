\doxysubsection{Detailed Description}
Abstract base class for dataseries that can be drawn on a \mbox{\hyperlink{classWt_1_1Chart_1_1WCartesian3DChart}{W\+Cartesian3\+D\+Chart}}. 

\doxyparagraph*{General}

The model that is provided at construction or with \mbox{\hyperlink{classWt_1_1Chart_1_1WAbstractDataSeries3D_a1b7fe4d6ae21b20546d020168dd53f63}{set\+Model()}} contains the data of a dataseries. Implementations of this class format the data for representation on the chart and perform all necessary drawing operations. \mbox{\hyperlink{classNote}{Note}} that if a dataseries holds numerical data it should be added to a chart of type \mbox{\hyperlink{group__charts_gga8d63464f873580c77508e1c0c26cbfeaa09870720ca8134284e4e305ac6ce5f19}{Chart\+Type\+::\+Scatter}}, if it holds categorical data it should be added to a chart of type \mbox{\hyperlink{group__charts_gga8d63464f873580c77508e1c0c26cbfeaa3adbdb3ac060038aa0e6e6c138ef9873}{Chart\+Type\+::\+Category}}.

\doxyparagraph*{Color}

The color used to draw data on a chart can be specified in a number of ways. The priority of this is as follows (1 being the highest)\+:


\begin{DoxyEnumerate}
\item \mbox{\hyperlink{classWt_1_1ItemDataRole_abb34e7ee20014dba0e303eab053d1cc5}{Item\+Data\+Role\+::\+Marker\+Brush\+Color}} set on a value in the model 
\item \mbox{\hyperlink{classWt_1_1Chart_1_1WAbstractColorMap}{W\+Abstract\+Color\+Map}} set on the dataseries 
\item \mbox{\hyperlink{classWt_1_1Chart_1_1WChartPalette}{W\+Chart\+Palette}} present in the chart 
\end{DoxyEnumerate}

A chart-\/palette will specify one color for the entire dataseries. Each new dataseries on a chart will receive another color.~\newline
 A colormap assigns different colors to the data within one dataseries, based on the z-\/value of the data. \mbox{\hyperlink{classWt_1_1Chart_1_1WStandardColorMap}{W\+Standard\+Color\+Map}} provides an easy way to create a colormap that is either smooth or consists of a number of bands.

\doxyparagraph*{Data-\/roles}

The roles on the model which are taken into account are\+:~\newline

\begin{DoxyItemize}
\item \mbox{\hyperlink{classWt_1_1ItemDataRole_abb34e7ee20014dba0e303eab053d1cc5}{Item\+Data\+Role\+::\+Marker\+Brush\+Color}}\+: this determines the color of a datapoint and overrides the default 
\item \mbox{\hyperlink{classWt_1_1ItemDataRole_a757e4e9ef3813b362eec115be1983178}{Item\+Data\+Role\+::\+Marker\+Scale\+Factor}}\+: this determines the size of a datapoint and overrides the default 
\end{DoxyItemize}

Some representations of the data ignore these roles. For example, when a surface is drawn, the roles are ignored.

\doxyparagraph*{Implementing a new dataseries class}

When the existing implementations of W\+Abstract\+Data\+Series3D don\textquotesingle{}t meet your needs, you might want to make your own. When doing this there are some details of the chart that you should know. The chart is made so that when a property of the chart changes, which affect any of the GL resources, all GL resources are destroyed and re-\/initialized. This eliminates the need to determine which chart-\/setting affect which G\+L-\/resources, which can be a complicated problem.

Therefore only unchanging GL resources are initialized in \mbox{\hyperlink{classWt_1_1Chart_1_1WAbstractDataSeries3D_a09f88c47c2fd01d83860dab188cad9b7}{initialize\+G\+L()}}. The initialize\+GL function in the chart is implemented to immediately request a call to \mbox{\hyperlink{classWt_1_1Chart_1_1WAbstractDataSeries3D_ab97b255bfc1c4af25da3828c82a8a8c7}{update\+G\+L()}}, which then initializes the rest of the GL resources. Every call to update\+GL in the chart, will first call \mbox{\hyperlink{classWt_1_1Chart_1_1WAbstractDataSeries3D_a49434192a0398755cec3bd1760450d46}{delete\+All\+G\+L\+Resources()}} on all dataseries and will then call \mbox{\hyperlink{classWt_1_1Chart_1_1WAbstractDataSeries3D_ab97b255bfc1c4af25da3828c82a8a8c7}{update\+G\+L()}} on all dataseries. So, when implementing a dataseries\+: initialize unchanging GL resources in \mbox{\hyperlink{classWt_1_1Chart_1_1WAbstractDataSeries3D_a09f88c47c2fd01d83860dab188cad9b7}{initialize\+G\+L()}}, initialize the rest of your GL resources in \mbox{\hyperlink{classWt_1_1Chart_1_1WAbstractDataSeries3D_ab97b255bfc1c4af25da3828c82a8a8c7}{update\+G\+L()}} and make G\+L-\/delete calls to all resources initialized in \mbox{\hyperlink{classWt_1_1Chart_1_1WAbstractDataSeries3D_ab97b255bfc1c4af25da3828c82a8a8c7}{update\+G\+L()}} in the function \mbox{\hyperlink{classWt_1_1Chart_1_1WAbstractDataSeries3D_a49434192a0398755cec3bd1760450d46}{delete\+All\+G\+L\+Resources()}}. It is also best to check is\+Null() on each of your G\+L-\/resources when deleting them.

This should be a local (server side) path to an image, such as a P\+NG or G\+IF. Only the alpha channel of this image is used\+: the sprite only decides if a pixel in the point appears or not. If the alpha value is below 0.\+5, the pixel is discarded.
