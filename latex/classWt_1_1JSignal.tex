\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename... A$>$\newline
class Wt\+::\+J\+Signal$<$ A $>$}

A signal to relay Java\+Script to C++ calls. 

A \mbox{\hyperlink{classWt_1_1JSignal}{J\+Signal}}, like an \mbox{\hyperlink{classWt_1_1EventSignal}{Event\+Signal}}, provides communicates events from Java\+Script to C++ code. However, it not tied to a built-\/in event. Instead, it can be emitted from within custom Java\+Script code using the Java\+Script Wt.\+emit() function.

The signal is identified by a unique name within the scope of a \mbox{\hyperlink{classWt_1_1WObject}{W\+Object}}, or a unique global name (when declaring the signal in your \mbox{\hyperlink{classWt_1_1WApplication}{W\+Application}}).

The signal supports up to 6 arguments. Values for these arguments may be specified in the Java\+Script Wt.\+emit().

Example code\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{class }MyWidget : \textcolor{keyword}{public} WCompositeWidget}
\DoxyCodeLine{\{}
\DoxyCodeLine{\textcolor{keyword}{public}:}
\DoxyCodeLine{  MyWidget()}
\DoxyCodeLine{    : doSome\_(this, \textcolor{stringliteral}{"doSome"})}
\DoxyCodeLine{  \{}
\DoxyCodeLine{    ...}
\DoxyCodeLine{  \}}
\DoxyCodeLine{}
\DoxyCodeLine{  JSignal<std::string, int>\& doSome() \{ \textcolor{keywordflow}{return} doSome\_; \}}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{keyword}{private}:}
\DoxyCodeLine{  JSignal<std::string, int> doSome\_;}
\DoxyCodeLine{}
\DoxyCodeLine{  ...}
\DoxyCodeLine{\};}
\end{DoxyCode}


The following Java\+Script statement will emit the signal for a D\+OM element {\itshape element} that corresponds to a widget of class {\ttfamily My\+Widget}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{namespaceWt}{Wt}}.emit(element, \textcolor{stringliteral}{'dosome'}, \textcolor{stringliteral}{'foo'}, 42);}
\end{DoxyCode}


The {\itshape element} can be a \mbox{\hyperlink{classWt_1_1WWidget_a83a186d2feacfbfbdb5b54f4cefc6735}{D\+OM element}}, or the \mbox{\hyperlink{classWt_1_1WObject_a9f34b0c9ec343cf467f8b237d3256792}{object ID}} of a \mbox{\hyperlink{classWt_1_1WObject}{W\+Object}}, or the constant {\ttfamily \mbox{\hyperlink{classWt}{Wt}}} which refers to the \mbox{\hyperlink{classWt_1_1WApplication}{Wt\+::\+W\+Application}} instance. The conversion between the Java\+Script arguments (ax) and the C++ type Ax uses stream operators.

You can use the methods \mbox{\hyperlink{classWt_1_1JSignal_ac7b6d3575d8443d4ae4ac85f5ed869e1}{create\+Call()}} to let the signal itself generate this Java\+Script call for you\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{doSome\_.createCall(\textcolor{stringliteral}{"'foo'"}, \textcolor{stringliteral}{"42"});}
\end{DoxyCode}


The Java\+Script generated by \mbox{\hyperlink{classWt_1_1JSignal_ac7b6d3575d8443d4ae4ac85f5ed869e1}{create\+Call()}} is possibly affected by every connect or disconnect to the signal. In practice, you will use \mbox{\hyperlink{classWt_1_1JSignal}{J\+Signal}} internally within a widget and call \mbox{\hyperlink{classWt_1_1JSignal_ac7b6d3575d8443d4ae4ac85f5ed869e1}{create\+Call()}} only after you connected internal slots to signal.

It is also possible to propagate an original Java\+Script event as a last argument, of type \mbox{\hyperlink{classWt_1_1WMouseEvent}{W\+Mouse\+Event}}, \mbox{\hyperlink{classWt_1_1WKeyEvent}{W\+Key\+Event}} or \mbox{\hyperlink{classWt_1_1WTouchEvent}{W\+Touch\+Event}}. In that case, the second argument in {\ttfamily Wt.\+emit()} must be an object which indicates also the Java\+Script event and event target.

Consider a signal declaration\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{JSignal<std::string, int, WMouseEvent> doSome();}
\end{DoxyCode}


Then, the following would be a suitable Java\+Script call\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{namespaceWt}{Wt}}.emit(\mbox{\hyperlink{namespaceWt}{Wt}}, \{\mbox{\hyperlink{classWt_1_1JSignal_a5d934e6626016915137fd0b01adc87e1}{name}}: \textcolor{stringliteral}{'dosome'}, event: event, eventObject: \textcolor{keywordtype}{object}\}, \textcolor{stringliteral}{'foo'}, 42);}
\end{DoxyCode}


The method \mbox{\hyperlink{classWt_1_1JSignal_a40e4e42f64060345e26ba85e612f217d}{create\+Event\+Call()}} may be used this variation for the Java\+Script method call.

Since the conversion from Java\+Script to C++ uses stream operators, you may provide support for custom types by implementing the c++ input stream operator {\ttfamily std\+::istream\& operator$>$$>$ (std\+::istream\&, T\& t)} for your type.

Construct a signal for the given object, and name. 

The given {\ttfamily name} must be unique for all user signals specified for the object {\ttfamily object}. Ownership of the signal is not transferred to the object.

If {\ttfamily collect\+Slot\+Java\+Script} is {\ttfamily true}, then javascript specified for connected slots (using \mbox{\hyperlink{classWt_1_1JSlot}{J\+Slot}}) or learned by stateless slot learning, is collected to client-\/side Java\+Script.

Use the utility methods \mbox{\hyperlink{classWt_1_1JSignal_ac7b6d3575d8443d4ae4ac85f5ed869e1}{create\+Call()}} or \mbox{\hyperlink{classWt_1_1JSignal_a40e4e42f64060345e26ba85e612f217d}{create\+Event\+Call()}} to create the appropriate Java\+Script statements that invoke the signal, which take into account possible other client-\/side Java\+Script handling associated with the signal.
