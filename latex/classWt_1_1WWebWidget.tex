\doxysubsection{Detailed Description}
A base class for widgets with an H\+T\+ML counterpart. 

All descendants of W\+Web\+Widget implement a widget which corresponds almost one-\/on-\/one with an H\+T\+ML element. These widgets provide most capabilities of these H\+T\+ML elements, but rarely make no attempt to do anything more.

Executes the given Java\+Script statements when the widget is rendered or updated. 

Calling \mbox{\hyperlink{classWt_1_1WApplication_a2a92457b9212cef4057cb54e56183967}{W\+Application\+::do\+Java\+Script()}} with Java\+Script code that refers to a widget using \mbox{\hyperlink{classWt_1_1WWidget_a83a186d2feacfbfbdb5b54f4cefc6735}{js\+Ref()}}, that is still to be rendered may cause Java\+Script errors because the corresponding D\+OM node does not exist. Resizes the widget. 

Specifies a fixed size for this widget, setting C\+SS {\ttfamily width} and {\ttfamily height} properties. By default a widget has automatic width and height, which sets a size for the widget following C\+SS rules.

When the widget is not managed by a layout manager, the automatic (natural) size of a widget depends on whether they widget is a {\itshape block} or {\itshape inline} widget\+:
\begin{DoxyItemize}
\item a {\itshape block} widget takes by default the width of the parent, and the height that it needs based on its contents
\item an {\itshape inline} widget takes the width and height that it needs based on its contents (possibly wrapping over multiple lines). The width and height of an inline widget cannot be changed (by the letter of C\+SS, although most browsers will react to it in varying ways).
\end{DoxyItemize}

When inserted in a layout manager, the size set will be used as a widget\textquotesingle{}s preferred size, but the widget may be given a different size by the layout manager based on available space and stretch factors. The actual size given by a layout manager may be retrieved by making the widget \char`\"{}layout size aware\char`\"{}, using \mbox{\hyperlink{classWt_1_1WWidget_a4f83592912a7f8fa4fd35dadde78ee74}{set\+Layout\+Size\+Aware()}}. If you have defined a {\ttfamily \char`\"{}wt\+Resize()\char`\"{}} Java\+Script method for the widget, then this method will also be called.

The default width and height of a widget is \mbox{\hyperlink{classWt_1_1WLength_a0cf39ca4225776879d56ade60320c31a}{W\+Length\+::\+Auto}}.

Disabled by default. When enabled, the client keeps track of whether this widget is currently visible inside of the browser window. A widget is \char`\"{}scroll visible\char`\"{} if it is currently visible according to \mbox{\hyperlink{classWt_1_1WWebWidget_a676dd25f88061270f7c6a84c2c202952}{is\+Visible()}}, and its position is inside of the browser window, with an extra margin determined by \mbox{\hyperlink{classWt_1_1WWebWidget_a4be8e1e673407f04eb2b98ea0487257d}{scroll\+Visibility\+Margin()}}.

If scroll visibility changes, the \mbox{\hyperlink{classWt_1_1WWebWidget_a3ac4848c8e5f9c038d37549bfe235b35}{scroll\+Visibility\+Changed()}} signal is fired, and \mbox{\hyperlink{classWt_1_1WWebWidget_a209077b839aedbf3c2443aec4de376d6}{is\+Scroll\+Visible()}} is updated.

This feature can be useful to implement infinite scroll, where a sentinel widget placed at the bottom of the page causes more content to be loaded when it becomes visible, see the {\ttfamily scrollvisibility} example.

This feature can also be used to lazy load widgets when they become visible.

\begin{DoxyNote}{Note}
If the widget is \char`\"{}scroll visible\char`\"{} when scroll visibility detection is first enabled, the \mbox{\hyperlink{classWt_1_1WWebWidget_a3ac4848c8e5f9c038d37549bfe235b35}{scroll\+Visibility\+Changed()}} signal will be emitted. If it is outside of the browser\textquotesingle{}s viewport when first enabled, the \mbox{\hyperlink{classWt_1_1WWebWidget_a3ac4848c8e5f9c038d37549bfe235b35}{scroll\+Visibility\+Changed()}} signal will not be emitted. 

If scroll visibility is enabled, disabled, and then enabled again, \mbox{\hyperlink{classWt_1_1WWebWidget_a209077b839aedbf3c2443aec4de376d6}{is\+Scroll\+Visible()}} may not be correctly updated, and \mbox{\hyperlink{classWt_1_1WWebWidget_a3ac4848c8e5f9c038d37549bfe235b35}{scroll\+Visibility\+Changed()}} may not be correctly emitted, because then \mbox{\hyperlink{classWt}{Wt}} can\textquotesingle{}t properly keep track of the state that the widget is in on the client side. This feature is not intended to be toggled on and off, but rather enabled once and disabled once after that. 
\end{DoxyNote}

Sets the tab index. 

For widgets that receive focus (\mbox{\hyperlink{classWt_1_1WWebWidget_a74d5aedc1f32c0c00c7d8905011cbd41}{can\+Receive\+Focus()}}), focus is passed on to the next widget in the {\itshape tabbing chain} based on their tab index. When the user navigates through form widgets using the keyboard, widgets receive focus starting from the element with the lowest tab index to elements with the highest tab index.

Widgets with a same tab index will receive focus in the same order as they are inserted in the widget tree.

The default tab index is 0 (for a widget that can receive focus).
