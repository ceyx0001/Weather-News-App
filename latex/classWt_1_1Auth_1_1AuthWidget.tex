\doxysubsection{Detailed Description}
An authentication widget. 

The authentication widget is a widget that provides a login or logout function (depending on whether the user is currently logged in). You can use it for either or both purposes.

\mbox{\hyperlink{classWt_1_1Auth_1_1Login}{Login}} or logout events are signalled to a \mbox{\hyperlink{classWt_1_1Auth_1_1Login}{Login}} object on which this widget acts.

The widget also processes environmental information related to authentication\+:


\begin{DoxyItemize}
\item email tokens, which are indicated in an internal path. The widget uses dialogs (by default) to interact with the user to act on the token.
\item authentication tokens, which are stored in browser cookies, to implement remember-\/me functionality.
\end{DoxyItemize}

The \mbox{\hyperlink{classWt_1_1Auth_1_1AuthWidget_ad8f632750848aff23e6b5e0cc339fd6a}{process\+Environment()}} method initiates this process, and should typically be called only at application startup time.

The authentication widget is implemented as a View for an \mbox{\hyperlink{classWt_1_1Auth_1_1AuthModel}{Auth\+Model}}, which can be set using \mbox{\hyperlink{classWt_1_1Auth_1_1AuthWidget_a93036a2d6cb7b39fa5101172591a717c}{set\+Model()}}. The login logic (at this moment only for password-\/based authentication) is handled by this model.

It is very likely that the off-\/the shelf authentication widget does not satisfy entirely to your taste or functional requirements. The widget uses three methods to allow customization\+:


\begin{DoxyItemize}
\item as a \mbox{\hyperlink{classWt_1_1WTemplateFormView}{W\+Template\+Form\+View}}, you may change the layout and styling of to your liking.
\item the authentication logic is delegated to an \mbox{\hyperlink{classWt_1_1Auth_1_1AuthModel}{Auth\+Model}} and can can be specialized or can be used with a custom view altogether.
\item the views are created using virtual methods, which may be specialized to create a customized view or to apply changes to the default view. 
\end{DoxyItemize}
