\doxysubsection{Detailed Description}
An model that manages a list of strings. 

This model only manages a unidimensional list of items and is optimized for usage by view widgets such as combo-\/boxes.

It supports all features of a typical item model, including data for multiple roles, editing and addition and removal of data rows.

You can populate the model by passing a list of strings to its consructor, or by using the \mbox{\hyperlink{classWt_1_1WStringListModel_a605688afa2a1810e1a01b3ef3d335bda}{set\+String\+List()}} method. You can set or retrieve data using the \mbox{\hyperlink{classWt_1_1WStringListModel_a2a1c0fd999847cd4b32fab690ff150a9}{set\+Data()}} and \mbox{\hyperlink{classWt_1_1WStringListModel_afc0d4b27b43b3e945190cfd8faaaf8c4}{data()}} methods, and add or remove data using the \mbox{\hyperlink{classWt_1_1WStringListModel_ab89a2d42280c92629d170b86f5e26ec8}{insert\+Rows()}} and \mbox{\hyperlink{classWt_1_1WStringListModel_a9bf4499ba3ff4f1489d962bc0d1d40d9}{remove\+Rows()}} methods.

Inserts one or more rows. 

In models that support row insertion, this inserts {\ttfamily count} rows, starting at {\ttfamily row}, and returns {\ttfamily true} if the operation was successful. The new rows are inserted under {\ttfamily parent}.

If parent had no children, then a single column is added with {\ttfamily count} rows.

The default implementation returns {\ttfamily false}.

The model implementation must call \mbox{\hyperlink{classWt_1_1WAbstractItemModel_a91e52898a805996aee496877cd090eb4}{begin\+Insert\+Rows()}} and \mbox{\hyperlink{classWt_1_1WAbstractItemModel_ace03c0bc5b0b57b6add882aaf444b62f}{end\+Insert\+Rows()}} before and after the operation whenever its geometry is changed by inserting rows. This emits signals for views to properly react to these changes.
