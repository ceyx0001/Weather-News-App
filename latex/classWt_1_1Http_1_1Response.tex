\doxysubsection{Detailed Description}
A resource response. 

This class defines the H\+T\+TP response for a \mbox{\hyperlink{classWt_1_1WResource}{W\+Resource}} request.

More specifically you can\+:
\begin{DoxyItemize}
\item set the content mime type using \mbox{\hyperlink{classWt_1_1Http_1_1Response_a20dd49d512ffc0cf407f9dd39db30341}{set\+Mime\+Type()}}
\item add H\+T\+TP headers using \mbox{\hyperlink{classWt_1_1Http_1_1Response_a5827dd04629251d0c7ff3cc9062afb7d}{add\+Header()}}
\item stream content into \mbox{\hyperlink{classWt_1_1Http_1_1Response_a5a76d6272200ead33695f22735341cd6}{out()}}
\end{DoxyItemize}

You may chose to provide only a partial response. In that case, use \mbox{\hyperlink{classWt_1_1Http_1_1Response_a76391b7300bdf3be3ca0a00f3670d592}{create\+Continuation()}} to create a continuation object to which you can annotate information for the next request to process the response further.

Create a continuation object for this response. 

A continuation is used to resume sending more data later for this response. There are two possible reasons for this\+:
\begin{DoxyItemize}
\item the entire response is quite big and you may want to read and send it in smaller chunks to avoid memory consumption problems since the I/O layer buffers the response first in memory to send it then out to a possibly slow client using async I/O.
\item you may not have any more data available, currently, but expect more data later. In that case you can call \mbox{\hyperlink{classWt_1_1Http_1_1ResponseContinuation_a3b441e3fa71a68b9dfb7091464e07433}{Response\+Continuation\+::wait\+For\+More\+Data()}} and later call \mbox{\hyperlink{classWt_1_1WResource_a1a9c35416eb690939ff288b053da3e1b}{W\+Resource\+::have\+More\+Data()}} when more data is available.
\end{DoxyItemize}

A new call to handle\+Request() will be made to retrieve more data.
