\doxysubsection{Detailed Description}
An abstract model for use with Wt\textquotesingle{}s view classes. 

This abstract model is used by several Wt view widgets as data models.

It may model data for both tree-\/like and table-\/like view widgets. \mbox{\hyperlink{classData}{Data}} is therefore organized in a hierarchical structure of tables, where every item stores data and items in column 0 can be the parent of a nested table of data. Every data item is uniquely identified by their row, column and parent index, and items may be referenced using the helper class \mbox{\hyperlink{classWt_1_1WModelIndex}{W\+Model\+Index}}.

Each item may provide data for one or more \mbox{\hyperlink{classWt_1_1ItemDataRole}{roles}}, and indicate options using \mbox{\hyperlink{group__modelview_ga1d77c4de293aebfacbdde124de06b8b0}{flags}}. The different roles can be used to model different aspects of an item (its text value, an icon, style class), or to hold auxiliary custom information. The flags provide information to the View on possible interactivity.

To implement a custom model, you need to reimplement the following methods\+:
\begin{DoxyItemize}
\item \mbox{\hyperlink{classWt_1_1WAbstractItemModel_a83f37bf04b8574b328c139b02523b8cb}{index()}} and \mbox{\hyperlink{classWt_1_1WAbstractItemModel_ada454d45706ca0c1a0921dedd351c48c}{parent()}} methods that allow one to navigate the model
\item \mbox{\hyperlink{classWt_1_1WAbstractItemModel_abcf7912a95a0a60f8e9b1c37d30b8383}{column\+Count()}} and \mbox{\hyperlink{classWt_1_1WAbstractItemModel_ab0709a8f71fd5df882f29bb29fd055f0}{row\+Count()}} to specify the top level geometry and the nested geometry at every item
\item \mbox{\hyperlink{classWt_1_1WAbstractItemModel_af77f707dd2340652717e7122cd7f4ffd}{data()}} to return the data for an item
\item optionally, \mbox{\hyperlink{classWt_1_1WAbstractItemModel_a37fa8987a50f8a10b5ffb123a84e3f3c}{header\+Data()}} to return row and column header data
\item optionally, \mbox{\hyperlink{classWt_1_1WAbstractItemModel_ad65561084d42facbde3e075801ed2e4d}{flags()}} to indicate data options
\end{DoxyItemize}

If you want to support editing of the model, then you need to indicate this support using a Wt\+::\+Item\+Flag\+::\+Editable flag, and reimplement \mbox{\hyperlink{classWt_1_1WAbstractItemModel_ab72f9eac35b1852e3b86273016a7f27e}{set\+Data()}}. View classes will use the \mbox{\hyperlink{classWt_1_1ItemDataRole_aa5ebe5a6363bc738d3efe4ad344e3aaf}{Item\+Data\+Role\+::\+Edit}} to read and update the data for the editor.

When the model\textquotesingle{}s data has been changed, the model must emit the \mbox{\hyperlink{classWt_1_1WAbstractItemModel_aa074e5bc55394252a609ef17d071b563}{data\+Changed()}} signal.

Finally, there is a generic interface for insertion of new data or removal of data (changing the geometry), although this interface is not yet used by any View class\+:


\begin{DoxyItemize}
\item \mbox{\hyperlink{classWt_1_1WAbstractItemModel_a59b5169e158b4cbfb1ec98df67c87200}{insert\+Rows()}}
\item \mbox{\hyperlink{classWt_1_1WAbstractItemModel_acbad59b06e90753f5e5e83de82ccd166}{insert\+Columns()}}
\item \mbox{\hyperlink{classWt_1_1WAbstractItemModel_ad1788e38e96df5795fc453f63d557ba2}{remove\+Rows()}}
\item \mbox{\hyperlink{classWt_1_1WAbstractItemModel_a18d68ce5450fb7e55f4f47d99c9e7414}{remove\+Columns()}}
\end{DoxyItemize}

Alternatively, you can provide your own A\+PI for changing the model. In either case it is important that you call the corresponding protected member functions which will emit the relevant signals so that views can adapt themselves to the new geometry.

This method is overloaded for handling drop events on top of items or drop events between items (see \mbox{\hyperlink{classWt_a46e584f830f2d8c9f5b16fcee7affc9b}{Wt\+::\+Drop\+Location}}). This overload handles drops on top of items, but note that due to historical reasons it will also insert the items in between when called with Drop\+Action\+::\+Move.

The location in the model is indicated by the {\ttfamily row} and {\ttfamily column} within the {\ttfamily parent} index. If {\ttfamily row} is -\/1, then the item is appended to the {\ttfamily parent}. Otherwise, the item is inserted at or copied over the indicated item (and subsequent rows). When {\ttfamily action} is a \mbox{\hyperlink{}{Drop\+Action\+::\+Move}}, the original items are deleted from the source model.

You may want to reimplement this method if you want to handle other mime-\/type data, or if you want to refine how the drop event of an item selection must be interpreted.
