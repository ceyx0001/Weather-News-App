\doxysubsection{Detailed Description}
Classes that implement Wt\textquotesingle{}s model/view system. 

\doxysubsubsection*{1. Models}

The library provides support for tabular, tree-\/like and tree-\/table like models. All of these implement \mbox{\hyperlink{classWt_1_1WAbstractItemModel}{W\+Abstract\+Item\+Model}}.

\doxyparagraph*{Concrete model implementations}

The following concrete model implementations are provided\+:
\begin{DoxyItemize}
\item \mbox{\hyperlink{classWt_1_1WStandardItemModel}{Wt\+::\+W\+Standard\+Item\+Model}}\+: a full general purpose model, which stores data in memory.
\item \mbox{\hyperlink{classWt_1_1WStringListModel}{Wt\+::\+W\+String\+List\+Model}}\+: a low-\/height single column model, useful for simple list views
\item \mbox{\hyperlink{classWt_1_1Dbo_1_1QueryModel}{Wt\+::\+Dbo\+::\+Query\+Model}}\+: a database query backed model
\end{DoxyItemize}

\doxyparagraph*{Proxy model implementations}

Proxy models are helper models which wrap around another model, to provide additional functionality on top of the existing model. The following propy models\+:


\begin{DoxyItemize}
\item \mbox{\hyperlink{classWt_1_1WSortFilterProxyModel}{Wt\+::\+W\+Sort\+Filter\+Proxy\+Model}}\+: provides sorting and filtering support
\item \mbox{\hyperlink{classWt_1_1WAggregateProxyModel}{Wt\+::\+W\+Aggregate\+Proxy\+Model}}\+: provides column aggregation (useful only for views that implement column aggregation such as \mbox{\hyperlink{classWt_1_1WAbstractItemView}{W\+Abstract\+Item\+View}}\textquotesingle{}s).
\end{DoxyItemize}

\doxyparagraph*{Abstract models}

Abstract models cannot be instantiated as such, but are the base point for custom model implementations\+:


\begin{DoxyItemize}
\item \mbox{\hyperlink{classWt_1_1WAbstractItemModel}{Wt\+::\+W\+Abstract\+Item\+Model}}\+: abstract base class of all models
\item \mbox{\hyperlink{classWt_1_1WAbstractTableModel}{Wt\+::\+W\+Abstract\+Table\+Model}}\+: abstract base class for tabular models
\item \mbox{\hyperlink{classWt_1_1WAbstractProxyModel}{Wt\+::\+W\+Abstract\+Proxy\+Model}}\+: abstract base class for proxy models.
\end{DoxyItemize}

\doxysubsubsection*{2. Views}

\doxyparagraph*{Item-\/based views}


\begin{DoxyItemize}
\item \mbox{\hyperlink{classWt_1_1WComboBox}{W\+Combo\+Box}}\+: a combo box
\item \mbox{\hyperlink{classWt_1_1WSelectionBox}{W\+Selection\+Box}}\+: a selection box
\item \mbox{\hyperlink{classWt_1_1WTableView}{W\+Table\+View}}\+: a table view (with editing support)
\item \mbox{\hyperlink{classWt_1_1WTreeView}{W\+Tree\+View}}\+: a tree(-\/table) view (with editing support)
\item \mbox{\hyperlink{classWt_1_1WSuggestionPopup}{W\+Suggestion\+Popup}}\+: an intelligent input-\/driven combo box
\end{DoxyItemize}

\doxyparagraph*{Graphical views}


\begin{DoxyItemize}
\item \mbox{\hyperlink{classWt_1_1Chart_1_1WCartesianChart}{Chart\+::\+W\+Cartesian\+Chart}}\+: 2D cartesian chart
\item \mbox{\hyperlink{classWt_1_1Chart_1_1WPieChart}{Chart\+::\+W\+Pie\+Chart}}\+: pie charts
\end{DoxyItemize}

\doxysubsubsection*{3. Helper classes}

\doxyparagraph*{Model indexes}

\mbox{\hyperlink{classWt_1_1WModelIndex}{W\+Model\+Index}} represents an index to an item of a \mbox{\hyperlink{classWt_1_1WAbstractItemModel}{W\+Abstract\+Item\+Model}}, identified by a row, column and parent node.

\doxyparagraph*{Item delegates}

Item delegates are used by \mbox{\hyperlink{classWt_1_1WTableView}{W\+Table\+View}} and \mbox{\hyperlink{classWt_1_1WTreeView}{W\+Tree\+View}} to render a single item and to provide editing support.

The abstract base class is \mbox{\hyperlink{classWt_1_1WAbstractItemDelegate}{W\+Abstract\+Item\+Delegate}}, and a default implementation is provided by \mbox{\hyperlink{classWt_1_1WItemDelegate}{W\+Item\+Delegate}}.

Interprets a cpp17\+::any as a number value. 

The conversion works as follows\+:
\begin{DoxyItemize}
\item A cpp17\+::any without a value, or a string that does not represent a number, is converted to a {\itshape NaN}.
\item A string is lexically casted to a double

\item \mbox{\hyperlink{classData}{Data}} of type \mbox{\hyperlink{classWt_1_1WTime}{W\+Time}} is converted to an integer number as the number of milliseconds since midnight.
\end{DoxyItemize}

Other types are converted according to \mbox{\hyperlink{structWt_1_1any__traits_ad294344c847931e451a60dc7e065a18a}{Wt\+::any\+\_\+traits$<$\+Type$>$\+::as\+Number()}}. Registers M\+V\+C-\/support for a type passed in a cpp17\+::any. 

By registering a type using this method, the global function \mbox{\hyperlink{group__modelview_ga2f96ad66a5212499e5b4d08609800a64}{Wt\+::as\+String()}}, which converts a cpp17\+::any to a string, will know how to interpret a cpp17\+::any holding a value of type {\ttfamily Type} using \mbox{\hyperlink{structWt_1_1any__traits_a4308aa7ddee4785e2d4a22c84aee4b49}{Wt\+::any\+\_\+traits$<$\+Type$>$\+::as\+String()}}. This gives Wt\textquotesingle{}s built-\/in standard View classes the ability to display \mbox{\hyperlink{classWt_1_1ItemDataRole_ae3ecc48a21764c9fce36a9be37ccc2c1}{Wt\+::\+Item\+Data\+Role\+::\+Display}} data of a \mbox{\hyperlink{classWt_1_1WAbstractItemModel}{Wt\+::\+W\+Abstract\+Item\+Model}} model.

Similarly, \mbox{\hyperlink{group__modelview_ga48a507458cc9f4754fe8b6459f4ad3d1}{Wt\+::as\+Number()}} uses \mbox{\hyperlink{structWt_1_1any__traits_ad294344c847931e451a60dc7e065a18a}{Wt\+::any\+\_\+traits$<$\+Type$>$\+::as\+Number()}} for use by \mbox{\hyperlink{classWt_1_1Chart_1_1WCartesianChart}{Wt\+::\+Chart\+::\+W\+Cartesian\+Chart}} and \mbox{\hyperlink{classWt_1_1Chart_1_1WPieChart}{Wt\+::\+Chart\+::\+W\+Pie\+Chart}} as numerical data.

The default implementation of Wt\+::any\+\_\+traits$<$\+Type$>$ converts a value {\ttfamily t} to a \mbox{\hyperlink{classWt_1_1WString}{Wt\+::\+W\+String}} by using std\+::ostream$<$$<$, expecting a U\+T\+F-\/8 string. This conversion thus relies on the std\+::ostream$<$$<$ operator to be overloaded for the type. You may want to specialize \mbox{\hyperlink{structWt_1_1any__traits}{Wt\+::any\+\_\+traits}} to provide a custom conversion for a type, or if you want to take into account the format string.

The following types are registered by Wt itself\+:
\begin{DoxyItemize}
\item strings of type \mbox{\hyperlink{classWt_1_1WString}{W\+String}} or std\+::string
\item \mbox{\hyperlink{classWt_1_1WDate}{W\+Date}}, \mbox{\hyperlink{classWt_1_1WTime}{W\+Time}}, \mbox{\hyperlink{classWt_1_1WDateTime}{W\+Date\+Time}}
\item standard C++ numeric types (int, double, etc...)
\item bool
\end{DoxyItemize}

This method is thread-\/safe, and it is not an error to register the same type multiple times.
