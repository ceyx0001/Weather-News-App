This function will return {\ttfamily nullptr} if no verification took place, Wt was compiled without S\+SL support, or the web server was configured without client S\+SL certificates.

Session-\/bound resources will probably not use this method, but rely on the validation done at the start of the session (see \mbox{\hyperlink{classWt_1_1Http_1_1Request_af4d09d8e5da790c1535d9db24da2a803}{ssl\+Info()}} in \mbox{\hyperlink{classWt_1_1WEnvironment}{W\+Environment}}). Static resources on the other hand don\textquotesingle{}t have an associated session, so using this method you can perform client authentication verification.

The object returned is owned by \mbox{\hyperlink{classWt_1_1Http_1_1Request}{Request}} and will be deleted when the \mbox{\hyperlink{classWt_1_1Http_1_1Request}{Request}} object is destroyed.
