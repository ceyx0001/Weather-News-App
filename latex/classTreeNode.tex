\doxysubsection{Detailed Description}
Example implementation of a single tree list node. 

{\itshape This is an example of a basic treelist implementation. As of version 1.\+1.\+8, a more flexible treenode implementation is included as part of the library\+: W\+Tree\+Node.}

A tree list is constructed by nesting \mbox{\hyperlink{classTreeNode}{Tree\+Node}} objects in a tree hierarchy.

A \mbox{\hyperlink{classTreeNode}{Tree\+Node}} has a label, and optionally a two-\/state label icon, which defines a different image depending on the state of the node (expanded or collapsed). When the node has any children, a child count is also indicated.

Next to the icons, two style classes determine the look of a \mbox{\hyperlink{classTreeNode}{Tree\+Node}}\+: the label has style \char`\"{}treenodelabel\char`\"{}, and the child count has as style \char`\"{}treenodechildcount\char`\"{}.

Use C\+SS nested selectors to apply different styles to different treenodes. For example, to style the treenode with style class \char`\"{}mynode\char`\"{}\+:

The behaviour of the tree node is to collapse all children when the node is expanded (this is similar to how most tree node implementations work).

The widget uses a number of images which must be available in an \char`\"{}icons/\char`\"{} folder (see the Wt treelist examples).

This widget is part of the Wt treelist example.

Construct a tree node with the given label. 

The label is formatted in a W\+Text with the given formatting. The label\+Icon (if not 0) will appear next to the label and its state will reflect the expand/collapse state of the node.

Optionally, a user\+Content widget may be associated with the node. When expanded, this widget will be shown below the widget, but above any of the children nodes.
