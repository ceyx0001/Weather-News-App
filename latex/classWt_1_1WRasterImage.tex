\doxysubsection{Detailed Description}
A paint device for rendering to a raster image. 

A W\+Raster\+Image paint device provides support for creating raster images (such as P\+NG or G\+IF images).

It implements two main use-\/cases\+:


\begin{DoxyItemize}
\item When used either in conjunction with a \mbox{\hyperlink{classWt_1_1WPainter}{W\+Painter}}, it supports vector graphics operations, and can be used to make a P\+NG or G\+IF version of a \mbox{\hyperlink{classWt_1_1WPaintedWidget}{W\+Painted\+Widget}}\textquotesingle{}s contents.
\item It also provides a low-\/level A\+PI to color individual pixels using \mbox{\hyperlink{classWt_1_1WRasterImage_a96888a696b1f98c2d36fda556595ab08}{set\+Pixel()}}, which directly sets the raster pixels.
\end{DoxyItemize}

On Linux, the rendering is provided by \href{http://www.graphicsmagick.org/}{\texttt{ Graphics\+Magick}}, and this class is included in the library only if {\ttfamily libgraphicsmagick} was found during the build of the library. On Windows, Direct2D is used for rendering, which means that no extra dependencies have to be installed.

If Wt is built to use {\ttfamily libpango} for font support, then text rendering is done using this library. On Windows, Direct\+Write is used for font support functions. In other cases, you may want to configure True\+Type font search directories using \mbox{\hyperlink{classWt_1_1WRasterImage_a227a9c6f057fb6335fa305a6f8510c55}{add\+Font\+Collection()}}.

You can use the image as a resource and specialize \mbox{\hyperlink{classWt_1_1WRasterImage_a19a55883c3052cb60859dab2b2a952b7}{handle\+Request()}} to paint the contents on the fly. Alternatively can also use \mbox{\hyperlink{classWt_1_1WResource_aaf6d964de1056c7c9404f88fa5e5a0de}{write()}} to serialize to an image file (std\+::ostream).

The latter usage is illustrated by the code below\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{classWt_1_1Chart_1_1WCartesianChart}{Wt::Chart::WCartesianChart}} *chart = ...}
\DoxyCodeLine{}
\DoxyCodeLine{Wt::WRasterImage pngImage(\textcolor{stringliteral}{"png"}, 600, 400);}
\DoxyCodeLine{\{}
\DoxyCodeLine{  \mbox{\hyperlink{classWt_1_1WPainter}{Wt::WPainter}} p(\&pngImage);}
\DoxyCodeLine{  chart-\/>paint(p);}
\DoxyCodeLine{\}}
\DoxyCodeLine{std::ofstream f(\textcolor{stringliteral}{"chart.png"}, std::ios::out | std::ios::binary);}
\DoxyCodeLine{pngImage.write(f);}
\end{DoxyCode}


This paint device has the following limitations\+:
\begin{DoxyItemize}
\item drop shadows are (not yet?) supported. 
\end{DoxyItemize}

If Wt has been configured to use {\ttfamily libpango}, then font matching and character selection is done by libpango, which is seeded with information on installed fonts by fontconfig. In that case, invocations for this method is ignored.
