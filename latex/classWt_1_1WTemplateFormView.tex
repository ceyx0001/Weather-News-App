\doxysubsection{Detailed Description}
A template-\/based View class for form models. 

This implements a View to be used in conjunction with \mbox{\hyperlink{classWt_1_1WFormModel}{W\+Form\+Model}} models to implement forms.

For each model field, it uses a number of conventional template placeholder variables to represent the label, editor, and validation messages in the template. For a field name \textquotesingle{}{\itshape field}\textquotesingle{}, we have\+:
\begin{DoxyItemize}
\item \textquotesingle{}{\itshape field}\textquotesingle{}\+: the actual (form) widget for viewing/editing the value
\item \textquotesingle{}{\itshape field}-\/label\textquotesingle{}\+: the label text
\item \textquotesingle{}{\itshape field}-\/info\textquotesingle{}\+: a text that contains help or validation messages
\item \textquotesingle{}if\+:{\itshape field}\textquotesingle{}\+: condition for the visibility of the field
\end{DoxyItemize}

A typical template uses blocks of the following-\/format (in the example below illustrated for a field \textquotesingle{}User\+Name\textquotesingle{})\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\$\{<if:UserName>\}}
\DoxyCodeLine{  <label for="\$\{id:UserName\}">\$\{UserName-\/label\}</label>}
\DoxyCodeLine{  \$\{UserName\} \$\{UserName-\/info\}}
\DoxyCodeLine{\$\{</if:UserName>\}}
\end{DoxyCode}


The View may render fields of more than one model, and does not necessarily need to render all information of each model. The latter can be achieved by either calling \mbox{\hyperlink{classWt_1_1WTemplateFormView_aeb4de961663b0d34813c29e118c0ceb9}{update\+View\+Field()}} and \mbox{\hyperlink{classWt_1_1WTemplateFormView_a7ca7be0de48bede84f47855180c2b830}{update\+Model\+Field()}} for individual model fields, or by hiding fields in the model that are not to be shown in the view.

The \mbox{\hyperlink{classWt_1_1WTemplateFormView_a94e9fa42bcf964c29919d970b530d286}{update\+View()}} method updates the view based on a model (e.\+g. to propagate changed values or validation feed-\/back), while the \mbox{\hyperlink{classWt_1_1WTemplateFormView_a405420e2132c64201dc472e072cf0346}{update\+Model()}} method updates the model with values entered in the View.

Creates or updates a field in the View. 

This will update or create and bind widgets in the template to represent the field. To create the form widget that implements the editing, it calls \mbox{\hyperlink{classWt_1_1WTemplateFormView_aada213f1df57b242320f2efae1503af8}{create\+Form\+Widget()}}.

The default behaviour interprets \mbox{\hyperlink{classWt_1_1WFormModel_a57c318f268ec74825b49068644a77c8c}{W\+Form\+Model\+::is\+Visible()}}, \mbox{\hyperlink{classWt_1_1WFormModel_a618f005896c66f9d276d4f1eac068aba}{W\+Form\+Model\+::is\+Read\+Only()}}, \mbox{\hyperlink{classWt_1_1WFormModel_a4d1f32594e074146ddebd1f4b579d8bc}{W\+Form\+Model\+::label()}} and \mbox{\hyperlink{classWt_1_1WFormModel_a6b31d2aa0b0aecd24ddc936a740b773b}{W\+Form\+Model\+::validator()}} to update the View, and calls \mbox{\hyperlink{classWt_1_1WTemplateFormView_ad1e55cca184836d07321fc20fb558b9a}{update\+View\+Value()}} to update the view value. If no form widget has been set for the given {\ttfamily field} using \mbox{\hyperlink{classWt_1_1WTemplateFormView_a3b151978b17554bfb11b40c9b169aff5}{set\+Form\+Widget()}}, then it calls \mbox{\hyperlink{classWt_1_1WTemplateFormView_aada213f1df57b242320f2efae1503af8}{create\+Form\+Widget()}} to try to create one.

It\textquotesingle{}s usually more convenient to reimplement \mbox{\hyperlink{classWt_1_1WTemplateFormView_ad1e55cca184836d07321fc20fb558b9a}{update\+View\+Value()}} to override specifically how the value from the model should be used to update the form widget. \mbox{\Hypertarget{classWt_1_1WTemplateFormView_ad1e55cca184836d07321fc20fb558b9a}\label{classWt_1_1WTemplateFormView_ad1e55cca184836d07321fc20fb558b9a}}
