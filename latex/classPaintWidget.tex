Initialize the GL state when the widget is first shown. 

\mbox{\hyperlink{classPaintWidget_ac159afb73949f18af2fa21afb53377fb}{initialize\+G\+L()}} is called when the widget is first rendered, and when the webglcontextrestored signal is fired. You can distinguish between the first initialization and context restoration using \mbox{\hyperlink{classWt_1_1WGLWidget_a95701ef1dc1da6a694da630daba35b5e}{restoring\+Context()}}. It usually creates most of the GL related state\+: shaders, V\+B\+Os, uniform locations, ...

If this state is to be updated during the lifetime of the widget, you should specialize the \mbox{\hyperlink{classWt_1_1WGLWidget_afef1be32851001b0c3836975e67d6ad5}{update\+G\+L()}} to accomodate for this.

{\footnotesize\ttfamily void Paint\+Widget\+::paint\+GL (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [virtual]}}



Update the client-\/side painting function. 

This method is invoked client-\/side when a repaint is required, i.\+e. when the \mbox{\hyperlink{classWt_1_1WGLWidget_ab44c8692c63dd82a2b5bcf70961319c5}{repaint\+Slot()}} (a Java\+Script-\/side J\+Slot) is triggered. Typical examples are\+: after mouse-\/based camera movements, after a timed update of a camera or an object\textquotesingle{}s position, after a resize event (\mbox{\hyperlink{classPaintWidget_a57b42ec8224c6065b8478782a6f1969b}{resize\+G\+L()}} will also be called then), after an animation event, ... In many cases, this function will be executed client-\/side many many times.

Using the GL functions from this class, you construct a scene. The implementation tracks all Java\+Script calls that need to be performed to draw the scenes, and will replay them verbatim on every trigger of the \mbox{\hyperlink{classWt_1_1WGLWidget_ab44c8692c63dd82a2b5bcf70961319c5}{repaint\+Slot()}}. There are a few mechanisms that may be employed to change what is rendered without updating the \mbox{\hyperlink{classPaintWidget_a8db0bbabdd863bdd8630c49b26f9b43f}{paint\+G\+L()}} cache\+: 
\begin{DoxyItemize}
\item Client-\/side matrices may be used to change camera viewpoints, manipilate separate object\textquotesingle{}s model transformation matrices, ...  
\item Shader sources can be updated without requiring the paint function to be renewed 
\end{DoxyItemize}

Updating the \mbox{\hyperlink{classPaintWidget_a8db0bbabdd863bdd8630c49b26f9b43f}{paint\+G\+L()}} cache is usually not too expensive; the V\+B\+Os, which are large in many cases, are already at the client side, while the \mbox{\hyperlink{classPaintWidget_a8db0bbabdd863bdd8630c49b26f9b43f}{paint\+G\+L()}} code only draws the V\+B\+Os. Of course, if you have to draw many separate objects, the \mbox{\hyperlink{classPaintWidget_a8db0bbabdd863bdd8630c49b26f9b43f}{paint\+G\+L()}} JS code may become large and updating is more expensive.

In order to update the \mbox{\hyperlink{classPaintWidget_a8db0bbabdd863bdd8630c49b26f9b43f}{paint\+G\+L()}} cache, call \mbox{\hyperlink{classWt_1_1WGLWidget_a18d874bacc0cf315da7c9eff2867314a}{repaint\+G\+L()}} with the P\+A\+I\+N\+T\+\_\+\+GL parameter, which will cause the invocation of this method.

Act on resize events. 

Usually, this method only contains functions to set the viewport and the projection matrix (as this is aspect ration dependent).

\mbox{\hyperlink{classPaintWidget_a57b42ec8224c6065b8478782a6f1969b}{resize\+G\+L()}} is rendered after initialize\+GL, and whenever widget is resized. After this method finishes, the widget is repainted with the cached client-\/side paint function.
