\doxysubsection{Detailed Description}
A widget that holds and manages child widgets. 

A W\+Container\+Widget acts as a container for child widgets. Child widgets may be added directly to the container or using a layout manager.

Use \mbox{\hyperlink{classWt_1_1WContainerWidget_a800886eb60c904c8d00978cb8ac8d158}{add\+Widget()}} or pass the container as constructor argument to a widget to directly add children to the container, without using a layout manager. In that case, C\+S\+S-\/based layout is used, and the resulting display is determined by properties of the children and the container. By default, a W\+Container\+Widget is displayed as a \mbox{\hyperlink{classWt_1_1WWidget_ac78e3af143883334c82031790c87416e}{block }} and manages its children within a rectangle. Inline child widgets are layed out in lines, wrapping around as needed, while block child widgets are stacked vertically. The container may add padding at the container edges using \mbox{\hyperlink{classWt_1_1WContainerWidget_ac9fb9045ce6bd06e5aa53ee8456c0033}{set\+Padding()}}, and provide alignment of contents using \mbox{\hyperlink{classWt_1_1WContainerWidget_aba13c9e62c74bcba53519ee9cf41f40a}{set\+Content\+Alignment()}}. A container is rendered by default using a H\+T\+ML {\ttfamily div} tag, but this may be changed to an H\+T\+ML {\ttfamily ul} or {\ttfamily ol} tag to make use of other C\+SS layout techniques, using \mbox{\hyperlink{classWt_1_1WContainerWidget_a4b1857ba8e8a677887cd20aa657fb77f}{set\+List()}}. In addition, specializations of this class as implemented by \mbox{\hyperlink{classWt_1_1WAnchor}{W\+Anchor}}, \mbox{\hyperlink{classWt_1_1WGroupBox}{W\+Group\+Box}}, \mbox{\hyperlink{classWt_1_1WStackedWidget}{W\+Stacked\+Widget}} and \mbox{\hyperlink{classWt_1_1WTableCell}{W\+Table\+Cell}} provide other alternative rendering of the container.

When setting the W\+Container\+Widget \mbox{\hyperlink{classWt_1_1WWidget_ac78e3af143883334c82031790c87416e}{inline }} the container only acts as a conceptual container, offering a common style to its children. Inline children are still layed out inline within the flow of the parent container of this container, as if they were inserted directly into that parent container. Block children are then not allowed (according to the H\+T\+ML specification).

To use a layout manager instead of C\+S\+S-\/based layout, use \mbox{\hyperlink{classWt_1_1WContainerWidget_a60e57a8860785d6b587a36b616cb5e12}{set\+Layout()}} or pass the container as constructor argument to a layout manager. In that case you should not define any padding for the container, and widgets and nested layout managers must be added to the layout manager, instead of to the container directly.

Usage example\+:

When using a layout manager, you need to carefully consider the alignment of the layout manager with respect to the container\+: when the container\textquotesingle{}s height is unconstrained (not specified explicitly using \mbox{\hyperlink{classWt_1_1WWebWidget_a5113267dfd734df27d4b1c76e7d23abb}{resize()}} or a style class, and the container is not included in a layout manager), you should pass Alignment\+Flag\+::\+Top to \mbox{\hyperlink{classWt_1_1WContainerWidget_a60e57a8860785d6b587a36b616cb5e12}{set\+Layout()}}.

\doxyparagraph*{C\+SS}

Depending on its configuration and usage, the widget corresponds to the following H\+T\+ML tags\+:
\begin{DoxyItemize}
\item By default, the widget corresponds to a {\ttfamily $<$div$>$} tag.
\item When configured with set\+Inline(true), the widget corresponds to a {\ttfamily $<$span$>$}.
\item When configured with set\+List(true), the widget corresponds to a {\ttfamily $<$ul$>$}.
\item When configured with set\+List(true, true), the widget corresponds to a {\ttfamily $<$ol$>$}.
\item When inserted into a container widget that \mbox{\hyperlink{classWt_1_1WContainerWidget_a8d2a337cc5fe06daa632098274dd8d2e}{is\+List()}}, the widget corresponds to a {\ttfamily $<$li$>$}.
\end{DoxyItemize}

This widget does not provide styling, and can be styled using inline or external C\+SS as appropriate.
