\doxysubsection{Detailed Description}
A widget that allows a file to be uploaded. 

This widget is displayed as a box in which a filename can be entered and a browse button.

Depending on availability of Java\+Script, the behaviour of the widget is different, but the A\+PI is designed in a way which facilitates a portable use.

When Java\+Script is available, the file will not be uploaded until \mbox{\hyperlink{classWt_1_1WFileUpload_ac17094f2fd6ae8cac1035161f7d80b9c}{upload()}} is called. This will start an asynchronous upload (and thus return immediately). When no Java\+Script is available, the file will be uploaded with the next click event. Thus, \mbox{\hyperlink{classWt_1_1WFileUpload_ac17094f2fd6ae8cac1035161f7d80b9c}{upload()}} has no effect -- the file will already be uploaded, and the corresponding signals will already be emitted. To test if \mbox{\hyperlink{classWt_1_1WFileUpload_ac17094f2fd6ae8cac1035161f7d80b9c}{upload()}} will start an upload, you may check using the \mbox{\hyperlink{classWt_1_1WFileUpload_ac0b34ef1ceb8b6c2836756720fa86068}{can\+Upload()}} call.

Thus, to properly use the widget, one needs to follow these rules\+: 
\begin{DoxyItemize}
\item Be prepared to handle the \mbox{\hyperlink{classWt_1_1WFileUpload_a09f835e5966c48312014b91a4b8b2652}{uploaded()}} or \mbox{\hyperlink{classWt_1_1WFileUpload_a943e9ea6fb2eb542362afea67d527205}{file\+Too\+Large()}} signals also when \mbox{\hyperlink{classWt_1_1WFileUpload_ac17094f2fd6ae8cac1035161f7d80b9c}{upload()}} was not called. 
\item Check using \mbox{\hyperlink{classWt_1_1WFileUpload_ac0b34ef1ceb8b6c2836756720fa86068}{can\+Upload()}} if \mbox{\hyperlink{classWt_1_1WFileUpload_ac17094f2fd6ae8cac1035161f7d80b9c}{upload()}} will schedule a new upload. if (!can\+Upload()) then \mbox{\hyperlink{classWt_1_1WFileUpload_ac17094f2fd6ae8cac1035161f7d80b9c}{upload()}} will not have any effect. if (\mbox{\hyperlink{classWt_1_1WFileUpload_ac0b34ef1ceb8b6c2836756720fa86068}{can\+Upload()}}), \mbox{\hyperlink{classWt_1_1WFileUpload_ac17094f2fd6ae8cac1035161f7d80b9c}{upload()}} will start a new file upload, which completes succesfully using an \mbox{\hyperlink{classWt_1_1WFileUpload_a09f835e5966c48312014b91a4b8b2652}{uploaded()}} signal or a \mbox{\hyperlink{classWt_1_1WFileUpload_a943e9ea6fb2eb542362afea67d527205}{file\+Too\+Large()}} signals gets emitted.  
\end{DoxyItemize}

The W\+File\+Upload widget must be hidden or deleted when a file is received. In addition it is wise to prevent the user from uploading the file twice as in the example below.

The uploaded file is automatically spooled to a local temporary file which will be deleted together with the \mbox{\hyperlink{classWt_1_1WFileUpload}{W\+File\+Upload}} widget, unless \mbox{\hyperlink{classWt_1_1WFileUpload_a541d634e9f9e546f59417682811f01ef}{steal\+Spooled\+File()}} is called.

W\+File\+Upload is an \mbox{\hyperlink{classWt_1_1WWidget_ac78e3af143883334c82031790c87416e}{inline }} widget.

\doxyparagraph*{C\+SS}

The file upload itself corresponds to a {\ttfamily $<$input type=\char`\"{}file\char`\"{}$>$} tag, but may be wrapped in a {\ttfamily $<$form$>$} tag for an Ajax session to implement the asynchronous upload action. This widget does not provide styling, and styling through C\+SS is not well supported across browsers.
