\doxysubsection{Detailed Description}
A view class that displays a model as a tree or tree table. 

The view displays data from a \mbox{\hyperlink{classWt_1_1WAbstractItemModel}{W\+Abstract\+Item\+Model}} in a tree or tree table. It provides incremental rendering, allowing the display of data models of any size efficiently, without excessive use of client-\/ or serverside resources.

The rendering (and editing) of items is handled by a \mbox{\hyperlink{classWt_1_1WAbstractItemDelegate}{W\+Abstract\+Item\+Delegate}}, by default it uses \mbox{\hyperlink{classWt_1_1WItemDelegate}{W\+Item\+Delegate}} which renders data of all predefined roles (see also \mbox{\hyperlink{classWt_1_1ItemDataRole}{Wt\+::\+Item\+Data\+Role}}), including text, icons, checkboxes, and tooltips.

The view may support editing of items, if the model indicates support (see the Wt\+::\+Item\+Flag\+::\+Editable flag). You can define triggers that initiate editing of an item using \mbox{\hyperlink{classWt_1_1WAbstractItemView_a93d2efc8642a6b9c55b35b7665a13da5}{set\+Edit\+Triggers()}}. The actual editing is provided by the item delegate (you can set an appropriate delegate for one column using \mbox{\hyperlink{classWt_1_1WAbstractItemView_a044f062bbc48a046fa7de64e557f805a}{set\+Item\+Delegate\+For\+Column()}}). Using \mbox{\hyperlink{classWt_1_1WAbstractItemView_a9a5506e031a514696c4050ccc1521b6c}{set\+Edit\+Options()}} you can customize if and how the view deals with multiple editors.

By default, all but the first columns are given a width of 150px, and the first column takes the remaining size. {\bfseries{\mbox{\hyperlink{classNote}{Note}} that this may have as consequence that the first column\textquotesingle{}s size is reduced to 0.}} \mbox{\hyperlink{structColumn}{Column}} widths of all columns, including the first column, can be set through the A\+PI method \mbox{\hyperlink{classWt_1_1WTreeView_a8560c58e4933aa767c27d56168db5d4d}{set\+Column\+Width()}}, and also by the user using handles provided in the header.

Optionally, the treeview may be configured so that the first column is always visible while scrolling through the other columns, which may be convenient if you wish to display a model with many columns. Use set\+Column1\+Fixed() to enable this behaviour.

If the model supports sorting (\mbox{\hyperlink{classWt_1_1WAbstractItemModel_ab044b92afa69282ea2e351bcdfd7c0c7}{W\+Abstract\+Item\+Model\+::sort()}}), such as the \mbox{\hyperlink{classWt_1_1WStandardItemModel}{W\+Standard\+Item\+Model}}, then you can enable sorting buttons in the header, using \mbox{\hyperlink{classWt_1_1WAbstractItemView_ae1c86a0f37f97f63aed3b00fd8ffad7c}{set\+Sorting\+Enabled()}}.

You can allow selection on row or item level (using \mbox{\hyperlink{classWt_1_1WAbstractItemView_ad59b47cf9a1c5a65c5adc3e1900ceb5c}{set\+Selection\+Behavior()}}), and selection of single or multiple items (using \mbox{\hyperlink{classWt_1_1WAbstractItemView_af6fcd9761e2f7a3b497d3563c849d320}{set\+Selection\+Mode()}}), and listen for changes in the selection using the \mbox{\hyperlink{classWt_1_1WAbstractItemView_ada37ffbe2d470dd227c8714645f150f0}{selection\+Changed()}} signal.

You may enable drag \& drop support for this view, with awareness of the items in the model. When enabling dragging (see \mbox{\hyperlink{classWt_1_1WAbstractItemView_a0ca99b835708d990b90000917045738e}{set\+Drag\+Enabled()}}), the current selection may be dragged, but only when all items in the selection indicate support for dragging (controlled by the \mbox{\hyperlink{}{Item\+Flag\+::\+Drag\+Enabled}} flag), and if the model indicates a mime-\/type (controlled by \mbox{\hyperlink{classWt_1_1WAbstractItemModel_a22a6c0dc9f4c9a6259a2daac07b78f61}{W\+Abstract\+Item\+Model\+::mime\+Type()}}). Likewise, by enabling support for dropping (see \mbox{\hyperlink{classWt_1_1WAbstractItemView_a1eda46c8e9a21af7f6281e7096af8fbe}{set\+Drops\+Enabled()}}), the treeview may receive a drop event on a particular item, at least if the item indicates support for drops (controlled by the \mbox{\hyperlink{}{Item\+Flag\+::\+Drop\+Enabled}} flag).

You may also react to mouse click events on any item, by connecting to one of the \mbox{\hyperlink{classWt_1_1WAbstractItemView_a3b72d5f8e5b0afdea83e224354d3e91b}{clicked()}} or \mbox{\hyperlink{classWt_1_1WAbstractItemView_a14ac099f58621d66834aa5674ea8ecb5}{double\+Clicked()}} signals.

\doxyparagraph*{Graceful degradation}

The view provides a virtual scrolling behavior which relies on Ajax availability. When Ajax is not available, a page navigation bar is used instead, see \mbox{\hyperlink{classWt_1_1WAbstractItemView_a5dcb8d0f45a3c9b1c9f238a533f77b51}{create\+Page\+Navigation\+Bar()}}. In that case, the widget needs to be given an explicit height using \mbox{\hyperlink{classWt_1_1WTreeView_afc6d07a4ee9c3a3d979ded4da4253c36}{resize()}} which determines the number of rows that are displayed at a time.

Resizes the widget. 

Specifies a fixed size for this widget, setting C\+SS {\ttfamily width} and {\ttfamily height} properties. By default a widget has automatic width and height, which sets a size for the widget following C\+SS rules.

When the widget is not managed by a layout manager, the automatic (natural) size of a widget depends on whether they widget is a {\itshape block} or {\itshape inline} widget\+:
\begin{DoxyItemize}
\item a {\itshape block} widget takes by default the width of the parent, and the height that it needs based on its contents
\item an {\itshape inline} widget takes the width and height that it needs based on its contents (possibly wrapping over multiple lines). The width and height of an inline widget cannot be changed (by the letter of C\+SS, although most browsers will react to it in varying ways).
\end{DoxyItemize}

When inserted in a layout manager, the size set will be used as a widget\textquotesingle{}s preferred size, but the widget may be given a different size by the layout manager based on available space and stretch factors. The actual size given by a layout manager may be retrieved by making the widget \char`\"{}layout size aware\char`\"{}, using \mbox{\hyperlink{classWt_1_1WWidget_a4f83592912a7f8fa4fd35dadde78ee74}{set\+Layout\+Size\+Aware()}}. If you have defined a {\ttfamily \char`\"{}wt\+Resize()\char`\"{}} Java\+Script method for the widget, then this method will also be called.
