\doxysubsection{Detailed Description}
An abstract widget that corresponds to an H\+T\+ML form element. 

A W\+Form\+Widget may receive focus (see \mbox{\hyperlink{classWt_1_1WFormWidget_a0389f78346d99b825fefdda83784efc5}{can\+Receive\+Focus()}}), can be disabled, and can have a label that acts as proxy for getting focus. It provides signals which reflect changes to its value, or changes to its focus.

\mbox{\hyperlink{namespaceWt_1_1Form}{Form}} widgets also have built-\/in support for validation, using \mbox{\hyperlink{classWt_1_1WFormWidget_a8b5fa8567e930691b95f1d0fa69359f1}{set\+Validator(const std\+::shared\+\_\+ptr$<$\+W\+Validator$>$ \&)}}. If the validator provide client-\/side validation, then an invalid validation state is reflected using the style class {\ttfamily \char`\"{}\+Wt-\/invalid\char`\"{}}. All validators provided by Wt implement client-\/side validation.

On the server-\/side, use \mbox{\hyperlink{classWt_1_1WFormWidget_ac895d6b344fa12d175b90f56f69ae0bb}{validate()}} method to validate the content using a validator previously set.
