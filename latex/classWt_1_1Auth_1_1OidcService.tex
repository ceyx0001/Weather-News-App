\doxysubsection{Detailed Description}
An Open\+Id Connect authentication service provider. 

This class implements an Open\+ID Connect client (\href{http://openid.net/specs/openid-connect-core-1_0.html}{\texttt{ core specification}}), which can be used to allow the user to be safely authenticated with your web application without needing to store or even handle his authorization credentials (such as a password).

Open\+ID Connect is a simple identity layer on top of the O\+Auth 2.\+0 protocol. It enables Clients to verify the identity of the End-\/\+User based on the authentication performed by an Authorization \mbox{\hyperlink{classWt_1_1Server}{Server}}, as well as to obtain basic profile information about the End-\/\+User in an interoperable and R\+E\+S\+T-\/like manner.

This implementation only supports authentication using the Authorization Code Flow.

The configuration of this service is done by using the setters the service class exposes. Before the authentication process can be started these settings must be configured first and may not be changed afterwards. 

The Open\+ID Connect protocol, including the subsequent use for authentication, consists of a number of consecutive steps, some of which require user interaction, and some which require the use of remote web services. The state machine for this process is implemented in an \mbox{\hyperlink{classWt_1_1Auth_1_1OidcProcess}{Oidc\+Process}}. To use Open\+ID Connect, you need to create such a process and listen for state changes.
