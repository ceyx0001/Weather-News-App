\doxysubsection{Detailed Description}
Abstract delegate class for rendering an item in an item view. 

Rendering of an item in a \mbox{\hyperlink{classWt_1_1WAbstractItemView}{W\+Abstract\+Item\+View}} is delegated to an implementation of this delegate class. The default implementation used by Wt\textquotesingle{}s item views is \mbox{\hyperlink{classWt_1_1WItemDelegate}{W\+Item\+Delegate}}. To provide specialized rendering support, you can reimplement this class (or specialize \mbox{\hyperlink{classWt_1_1WItemDelegate}{W\+Item\+Delegate}}).

As a delegate is used for rendering multiple items, the class should not keep state about one specific item.

A delegate may provide editing support by instantiating an editor when \mbox{\hyperlink{classWt_1_1WAbstractItemDelegate_a3d1735427e5dac683a19d156536db12d}{update()}} is called with the Wt\+::\+View\+Item\+Render\+Flag\+::\+Editing flag. In that case, you will also need to implement \mbox{\hyperlink{classWt_1_1WAbstractItemDelegate_a20d34d31faf8fdb961810d18449f05e7}{edit\+State()}} and \mbox{\hyperlink{classWt_1_1WAbstractItemDelegate_a8400deb4367c1cd1ca142a4714bcce14}{set\+Edit\+State()}} to support virtual scrolling and \mbox{\hyperlink{classWt_1_1WAbstractItemDelegate_a630f12863aeb4ed7f4e29d4a95bb7a07}{set\+Model\+Data()}} to save the edited value to the model. For an example, see the \mbox{\hyperlink{classWt_1_1WItemDelegate}{W\+Item\+Delegate}}.

Returns the current edit state. 

Because a View may support virtual scrolling in combination with editing, it may happen that the view decides to delete the editor widget while the user is editing. To allow to reconstruct the editor in its original state, the View will therefore ask for the editor to serialize its state in a boost\+::any.

When the view decides to close an editor and save its value back to the model, he will first call \mbox{\hyperlink{classWt_1_1WAbstractItemDelegate_a20d34d31faf8fdb961810d18449f05e7}{edit\+State()}} and then \mbox{\hyperlink{classWt_1_1WAbstractItemDelegate_a630f12863aeb4ed7f4e29d4a95bb7a07}{set\+Model\+Data()}}.

The default implementation assumes a read-\/only delegate, and returns a boost\+::any().

Creates or updates a widget that renders an item. 

The item is specified by its model {\ttfamily index}, which also indicates the model. If an existing widget already renders the item, but needs to be updated, it is passed as the {\ttfamily widget} parameter.

When {\ttfamily widget} is {\ttfamily nullptr}, a new widget needs to be created and returned.

If you want to replace the {\ttfamily widget} with a new one, return the new widget. The old {\ttfamily widget} will be removed. Return {\ttfamily nullptr} if you do not want to replace the {\ttfamily widget}.

You can remove the {\ttfamily widget} from its parent for reuse with \mbox{\hyperlink{classWt_1_1WWidget_a5331409ab84a19b6e6461a3f3f387aa6}{W\+Widget\+::remove\+From\+Parent()}}.

The returned widget should be a widget that responds properly to be given a height, width and style class. In practice, that means it cannot have a border or margin, and thus cannot be a \mbox{\hyperlink{classWt_1_1WFormWidget}{W\+Form\+Widget}} since those widgets typically have built-\/in borders and margins. If you want to return a form widget (for editing the item), you should wrap it in a container widget.

The {\ttfamily flags} parameter indicates options for rendering the item.
